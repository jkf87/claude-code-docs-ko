웹에서 작업 예약하기 ​

예약된 작업은 Anthropic이 관리하는 인프라를 사용해 반복 주기마다 프롬프트를 실행합니다. 내 컴퓨터가 꺼져 있어도 작업은 계속 실행됩니다.

자동화할 수 있는 반복 작업의 예시입니다:

  • 매일 아침 열린 풀 리퀘스트 검토
  • 밤 사이 CI 실패 분석 후 요약 제공
  • PR 병합 후 문서 동기화
  • 매주 의존성 감사 실행

예약된 작업은 Pro, Max, Team, Enterprise를 포함한 모든 Claude Code 웹 사용자에게 제공됩니다.

예약 옵션 비교 ​

Claude Code는 반복 작업을 예약하는 세 가지 방법을 제공합니다:

클라우드데스크탑/loop
실행 위치Anthropic 클라우드내 컴퓨터내 컴퓨터
컴퓨터 켜짐 필요아니오예예
열린 세션 필요아니오아니오예
재시작 후 지속성예예아니오 (세션 범위)
로컬 파일 접근아니오 (새로 클론)예예
MCP 서버작업별로 설정된 커넥터설정 파일 및 커넥터세션에서 상속
권한 프롬프트아니오 (자율 실행)작업별로 설정 가능세션에서 상속
예약 커스터마이즈CLI에서 /schedule 사용예예
최소 간격1시간1분1분

TIP

컴퓨터 없이도 안정적으로 실행해야 하는 작업에는 클라우드 작업을 사용하세요. 로컬 파일과 도구에 접근해야 할 때는 데스크탑 작업을 사용하세요. 세션 중 빠른 폴링에는 **/loop**를 사용하세요.

예약된 작업 만들기 ​

예약된 작업은 세 곳에서 만들 수 있습니다:

  • 웹: claude.ai/code/scheduled에 방문해 새 예약 작업을 클릭하세요.
  • 데스크탑 앱: Schedule 페이지를 열고 New task를 클릭한 뒤 New remote task를 선택하세요. 자세한 내용은 데스크탑 예약 작업을 참고하세요.
  • CLI: 아무 세션에서나 /schedule을 실행하세요. Claude가 대화형으로 설정을 안내합니다. /schedule daily PR review at 9am처럼 설명을 직접 전달할 수도 있습니다.

웹과 데스크탑 진입점은 양식을 열어줍니다. CLI는 안내 대화를 통해 동일한 정보를 수집합니다.

아래 단계는 웹 인터페이스 사용 방법을 안내합니다.

1단계: 생성 양식 열기

claude.ai/code/scheduled에 방문해 새 예약 작업을 클릭하세요.

2단계: 작업 이름 지정 및 프롬프트 작성

작업에 설명적인 이름을 지정하고 Claude가 매번 실행할 프롬프트를 작성하세요. 프롬프트가 가장 중요한 부분입니다. 작업은 자율적으로 실행되므로 프롬프트는 독립적이어야 하고, 무엇을 해야 하는지와 성공의 기준이 명확해야 합니다.

프롬프트 입력 영역에는 모델 선택기가 포함되어 있습니다. Claude는 작업의 각 실행에 이 모델을 사용합니다.

3단계: 저장소 선택

Claude가 작업할 하나 이상의 GitHub 저장소를 추가하세요. 각 저장소는 실행 시작 시 기본 브랜치에서 클론됩니다. Claude는 변경 사항에 대해 claude/ 접두사가 붙은 브랜치를 생성합니다. 모든 브랜치로의 푸시를 허용하려면 해당 저장소에서 Allow unrestricted branch pushes를 활성화하세요.

4단계: 환경 선택

작업을 위한 클라우드 환경을 선택하세요. 환경은 클라우드 세션이 접근할 수 있는 항목을 제어합니다:

  • 네트워크 접근: 각 실행 중 사용 가능한 인터넷 접근 수준 설정
  • 환경 변수: Claude가 사용할 API 키, 토큰, 또는 기타 시크릿 제공
  • 설정 스크립트: 각 세션 시작 전 의존성 설치나 도구 설정 같은 설치 명령 실행

Default 환경은 기본으로 제공됩니다. 커스텀 환경을 사용하려면 작업을 만들기 전에 환경을 생성하세요.

5단계: 예약 선택

빈도 옵션에서 작업 실행 빈도를 선택하세요. 기본값은 현지 시간대 기준 매일 오전 9시입니다. 지연을 분산시키기 위해 작업은 예약된 시간에서 몇 분 후에 실행될 수 있습니다.

사전 설정 옵션이 맞지 않으면 가장 가까운 옵션을 선택한 후 CLI에서 /schedule update로 예약을 업데이트해 특정 일정을 설정하세요.

6단계: 커넥터 검토

연결된 모든 MCP 커넥터가 기본으로 포함됩니다. 작업에 필요하지 않은 커넥터는 제거하세요. 커넥터를 통해 Claude는 각 실행 중 Slack, Linear, Google Drive 같은 외부 서비스에 접근할 수 있습니다.

7단계: 작업 생성

Create를 클릭하세요. 작업이 예약된 작업 목록에 나타나고 다음 예약 시간에 자동으로 실행됩니다. 각 실행은 다른 세션과 함께 새 세션을 생성하며, Claude가 무엇을 했는지 확인하고, 변경 사항을 검토하고, 풀 리퀘스트를 생성할 수 있습니다. 즉시 실행을 트리거하려면 작업 상세 페이지에서 Run now를 클릭하세요.

빈도 옵션 ​

예약 선택기는 시간대 변환을 자동으로 처리하는 사전 설정 빈도를 제공합니다. 현지 시간대로 시간을 선택하면 클라우드 인프라가 어디에 위치하든 해당 시각에 작업이 실행됩니다.

:::note 작업은 예약된 시간에서 몇 분 후에 실행될 수 있습니다. 오프셋은 각 작업에 대해 일관적입니다. :::

빈도설명
Hourly매 시간마다 실행됩니다.
Daily지정한 시간에 하루 한 번 실행됩니다. 기본값은 현지 시간 오전 9시입니다.
WeekdaysDaily와 동일하지만 토요일과 일요일은 건너뜁니다.
Weekly지정한 요일과 시간에 주 1회 실행됩니다.

2시간마다 또는 매월 1일 같은 커스텀 간격의 경우 가장 가까운 사전 설정을 선택하고 CLI에서 /schedule update로 예약을 업데이트해 특정 cron 표현식을 설정하세요. 최소 간격은 1시간입니다. */30 * * * *처럼 더 자주 실행되는 표현식은 거부됩니다.

저장소 및 브랜치 권한 ​

예약된 작업이 저장소를 클론하려면 GitHub 접근 권한이 필요합니다. CLI에서 /schedule로 작업을 만들 때 Claude는 계정에 GitHub가 연결되어 있는지 확인하고, 연결되어 있지 않으면 /web-setup을 실행하도록 안내합니다. 접근 권한을 부여하는 두 가지 방법은 GitHub 인증 옵션을 참고하세요.

추가한 각 저장소는 실행마다 클론됩니다. 프롬프트에서 별도로 지정하지 않으면 Claude는 저장소의 기본 브랜치에서 시작합니다.

기본적으로 Claude는 claude/ 접두사가 붙은 브랜치에만 푸시할 수 있습니다. 이는 예약된 작업이 보호된 브랜치나 장기 브랜치를 실수로 수정하는 것을 방지합니다.

특정 저장소에 대해 이 제한을 해제하려면 작업을 만들거나 편집할 때 해당 저장소에서 Allow unrestricted branch pushes를 활성화하세요.

커넥터 ​

예약된 작업은 연결된 MCP 커넥터를 사용해 각 실행 중 외부 서비스를 읽고 쓸 수 있습니다. 예를 들어 지원 요청을 분류하는 작업은 Slack 채널을 읽고 Linear에 이슈를 생성할 수 있습니다.

작업을 만들 때 현재 연결된 모든 커넥터가 기본으로 포함됩니다. 실행 중 Claude가 접근하는 도구를 제한하기 위해 필요하지 않은 커넥터는 제거하세요. 작업 양식에서 직접 커넥터를 추가할 수도 있습니다.

작업 양식 외부에서 커넥터를 관리하거나 추가하려면 claude.ai의 Settings > Connectors를 방문하거나 CLI에서 /schedule update를 사용하세요.

환경 ​

각 작업은 네트워크 접근, 환경 변수, 설정 스크립트를 제어하는 클라우드 환경에서 실행됩니다. Claude가 API에 접근하고, 의존성을 설치하고, 네트워크 범위를 제한할 수 있도록 작업을 만들기 전에 환경을 설정하세요. 전체 설정 가이드는 클라우드 환경을 참고하세요.

예약된 작업 관리 ​

Scheduled 목록에서 작업을 클릭하면 상세 페이지가 열립니다. 상세 페이지에는 작업의 저장소, 커넥터, 프롬프트, 예약, 그리고 지난 실행 목록이 표시됩니다.

실행 보기 및 상호작용 ​

실행 항목을 클릭하면 전체 세션으로 열립니다. 여기서 Claude가 무엇을 했는지 확인하고, 변경 사항을 검토하고, 풀 리퀘스트를 생성하거나, 대화를 계속할 수 있습니다. 각 실행 세션은 다른 세션과 동일하게 작동합니다. 세션 제목 옆의 드롭다운 메뉴를 사용해 이름 변경, 보관, 삭제가 가능합니다.

작업 편집 및 제어 ​

작업 상세 페이지에서 다음을 수행할 수 있습니다:

  • Run now를 클릭해 다음 예약 시간을 기다리지 않고 즉시 실행을 시작합니다.
  • Repeats 섹션의 토글을 사용해 예약을 일시 중지하거나 재개합니다. 일시 중지된 작업은 설정을 유지하지만 다시 활성화할 때까지 실행되지 않습니다.
  • 편집 아이콘을 클릭해 이름, 프롬프트, 예약, 저장소, 환경, 또는 커넥터를 변경합니다.
  • 삭제 아이콘을 클릭해 작업을 제거합니다. 작업이 생성한 지난 세션은 세션 목록에 남아 있습니다.

CLI에서 /schedule로도 작업을 관리할 수 있습니다. /schedule list로 모든 작업 목록 보기, /schedule update로 작업 변경, /schedule run으로 즉시 실행 트리거가 가능합니다.

관련 자료 ​

  • 데스크탑 예약 작업: 로컬 파일에 접근하며 내 컴퓨터에서 실행되는 작업 예약. 데스크탑 앱의 Schedule 페이지에서 로컬 및 원격 작업을 같은 그리드에서 볼 수 있습니다.
  • /loop 및 CLI 예약 작업: CLI 세션 내 경량 예약
  • 클라우드 환경: 클라우드 작업의 런타임 환경 설정
  • MCP 커넥터: Slack, Linear, Google Drive 같은 외부 서비스 연결
  • GitHub Actions: 저장소 이벤트에 따라 CI 파이프라인에서 Claude 실행
