# Agent SDK 호스팅

> 프로덕션 환경에서 Claude Agent SDK 배포 및 호스팅

Claude Agent SDK는 기존의 무상태(stateless) LLM API와 달리, 대화 상태를 유지하고 지속적인 환경에서 명령을 실행합니다. 이 가이드에서는 SDK 기반 에이전트를 프로덕션에 배포하기 위한 아키텍처, 호스팅 고려 사항, 그리고 모범 사례를 다룹니다.

::: info
기본 샌드박싱을 넘어선 보안 강화(네트워크 제어, 자격 증명 관리, 격리 옵션 포함)에 대해서는 [보안 배포](/agent-sdk/secure-deployment)를 참고하세요.
:::

## 호스팅 요구 사항

### 컨테이너 기반 샌드박싱

보안과 격리를 위해 SDK는 샌드박스 컨테이너 환경 내에서 실행해야 합니다. 이를 통해 프로세스 격리, 리소스 제한, 네트워크 제어, 임시 파일 시스템을 확보할 수 있습니다.

SDK는 명령 실행을 위한 [프로그래밍 방식 샌드박스 설정](/agent-sdk/typescript#sandbox-settings)도 지원합니다.

### 시스템 요구 사항

각 SDK 인스턴스에는 다음이 필요합니다:

* **런타임 의존성**
  * Python 3.10+ (Python SDK 사용 시) 또는 Node.js 18+ (TypeScript SDK 사용 시)
  * Node.js (SDK가 내부적으로 실행하는 Claude Code CLI에 필요; 두 SDK 패키지에 모두 포함되어 있어 별도 설치 불필요)

* **리소스 할당**
  * 권장: RAM 1GiB, 디스크 5GiB, CPU 1개 (작업에 따라 조정 가능)

* **네트워크 접근**
  * `api.anthropic.com`으로의 아웃바운드 HTTPS
  * 선택 사항: MCP 서버 또는 외부 도구 접근

## SDK 아키텍처 이해하기

무상태 API 호출과 달리, Claude Agent SDK는 다음 기능을 갖춘 **장기 실행 프로세스**로 동작합니다:

* **명령 실행** - 지속적인 셸 환경에서 명령을 실행
* **파일 작업 관리** - 작업 디렉토리 내에서 파일 작업 처리
* **도구 실행 처리** - 이전 상호작용의 컨텍스트를 활용한 도구 실행

## 샌드박스 제공업체 옵션

AI 코드 실행을 위한 안전한 컨테이너 환경을 전문으로 제공하는 업체들입니다:

* **[Modal Sandbox](https://modal.com/docs/guide/sandbox)** - [데모 구현 예시](https://modal.com/docs/examples/claude-slack-gif-creator)
* **[Cloudflare Sandboxes](https://github.com/cloudflare/sandbox-sdk)**
* **[Daytona](https://www.daytona.io/)**
* **[E2B](https://e2b.dev/)**
* **[Fly Machines](https://fly.io/docs/machines/)**
* **[Vercel Sandbox](https://vercel.com/docs/functions/sandbox)**

자체 호스팅 옵션(Docker, gVisor, Firecracker)과 상세한 격리 설정에 대해서는 [격리 기술](/agent-sdk/secure-deployment#isolation-technologies)을 참고하세요.

## 프로덕션 배포 패턴

### 패턴 1: 임시(Ephemeral) 세션

사용자 작업마다 새 컨테이너를 생성하고, 작업이 완료되면 삭제합니다.

일회성 작업에 적합합니다. 사용자는 작업이 완료되는 동안 AI와 계속 상호작용할 수 있지만, 작업이 완료되면 컨테이너는 삭제됩니다.

**활용 예시:**

* 버그 조사 및 수정: 관련 컨텍스트를 활용해 특정 이슈를 디버그하고 해결
* 청구서 처리: 영수증/청구서에서 데이터를 추출해 회계 시스템에 맞게 구조화
* 번역 작업: 문서나 콘텐츠 묶음을 언어 간 번역
* 이미지/비디오 처리: 미디어 파일에 변환, 최적화 적용 또는 메타데이터 추출

### 패턴 2: 장기 실행 세션

장기 작업을 위해 지속적인 컨테이너 인스턴스를 유지합니다. 수요에 따라 컨테이너 내부에서 *여러* Claude Agent 프로세스를 실행하는 경우가 많습니다.

사용자 입력 없이 자율적으로 행동하는 능동적 에이전트, 콘텐츠를 제공하는 에이전트, 또는 대량의 메시지를 처리하는 에이전트에 적합합니다.

**활용 예시:**

* 이메일 에이전트: 수신 이메일을 모니터링하고 내용에 따라 자율적으로 분류, 응답 또는 작업 수행
* 사이트 빌더: 사용자별 맞춤 웹사이트를 호스팅하고 컨테이너 포트를 통해 실시간 편집 기능 제공
* 고빈도 챗봇: Slack과 같은 플랫폼에서 빠른 응답 시간이 중요한 연속적인 메시지 스트림 처리

### 패턴 3: 하이브리드 세션

데이터베이스나 SDK의 세션 재개 기능을 통해 기록과 상태가 복원되는 임시 컨테이너입니다.

사용자가 간헐적으로 상호작용하며 작업을 시작하고, 작업이 완료되면 종료되지만 이후에도 계속할 수 있는 컨테이너에 적합합니다.

**활용 예시:**

* 개인 프로젝트 관리자: 간헐적인 체크인으로 진행 중인 프로젝트를 관리하고, 작업, 결정, 진행 상황의 컨텍스트를 유지
* 심층 리서치: 몇 시간에 걸친 리서치 작업을 수행하고, 결과를 저장한 후 사용자가 돌아올 때 조사를 재개
* 고객 지원 에이전트: 여러 번의 상호작용에 걸친 지원 티켓을 처리하고, 티켓 기록과 고객 컨텍스트를 불러옴

### 패턴 4: 단일 컨테이너

하나의 전역 컨테이너에서 여러 Claude Agent SDK 프로세스를 실행합니다.

긴밀히 협업해야 하는 에이전트에 적합합니다. 에이전트들이 서로 덮어쓰지 않도록 방지해야 하기 때문에 가장 덜 사용되는 패턴입니다.

**활용 예시:**

* **시뮬레이션**: 비디오 게임과 같은 시뮬레이션에서 서로 상호작용하는 에이전트

## FAQ

### 샌드박스와 어떻게 통신하나요?

컨테이너에서 호스팅할 때, SDK 인스턴스와 통신하기 위해 포트를 노출합니다. 애플리케이션은 외부 클라이언트를 위한 HTTP/WebSocket 엔드포인트를 노출하고, SDK는 컨테이너 내부에서 실행될 수 있습니다.

### 컨테이너 호스팅 비용은 얼마인가요?

에이전트 서비스의 주요 비용은 토큰입니다. 컨테이너 비용은 프로비저닝 사양에 따라 다르지만, 최소 비용은 실행 시간당 약 5센트 수준입니다.

### 유휴 컨테이너를 언제 종료하고 언제 유지해야 하나요?

이는 제공업체에 따라 다릅니다. 다양한 샌드박스 제공업체마다 유휴 타임아웃 기준을 다르게 설정할 수 있으며, 그 후 샌드박스가 종료될 수 있습니다. 사용자의 응답 빈도를 예상하여 이 타임아웃을 조정하는 것이 좋습니다.

### Claude Code CLI는 얼마나 자주 업데이트해야 하나요?

Claude Code CLI는 semver로 버전 관리되므로 Breaking changes는 버전에 명시됩니다.

### 컨테이너 상태와 에이전트 성능을 어떻게 모니터링하나요?

컨테이너는 단순히 서버이므로, 백엔드에 사용하는 동일한 로깅 인프라가 컨테이너에도 적용됩니다.

### 에이전트 세션은 타임아웃 전까지 얼마나 실행될 수 있나요?

에이전트 세션은 타임아웃되지 않습니다. 다만 Claude가 루프에 갇히는 것을 방지하기 위해 `maxTurns` 속성 설정을 고려하세요.
