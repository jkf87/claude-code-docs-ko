# Claude Code 확장하기

> CLAUDE.md, Skills, 서브에이전트, Hooks, MCP, 플러그인을 언제 사용해야 하는지 이해하기

Claude Code는 코드에 대해 추론하는 모델과 [내장 도구](/how-claude-code-works#tools)를 결합한 파일 조작, 검색, 실행, 웹 접근을 위한 도구들을 제공합니다. 내장 도구는 대부분의 코딩 작업을 다룹니다. 이 가이드에서는 확장 레이어: Claude가 알 수 있는 기능을 추가하고, 외부 서비스에 연결하고, 워크플로우를 자동화하는 방법들을 다룹니다.

::: tip
코어 에이전트 루프가 동작하는 방법에 대해서는 [Claude Code 작동 원리](/how-claude-code-works)를 참조하세요.
:::

## 개요

확장은 에이전트 루프의 여러 부분에 연결됩니다:

* **[CLAUDE.md](/memory)** — 모든 세션에서 로드되는 지속 컨텍스트
* **[Skills](/skills)** — 재사용 가능한 지식과 호출 가능한 워크플로우
* **[MCP](/mcp)** — Claude를 외부 서비스 및 도구에 연결
* **[서브에이전트](/sub-agents)** — 고립 컨텍스트에서 자체 루프를 실행하고 요약된 결과 반환
* **[에이전트 팀](/agent-teams)** — 여러 독립 Claude Code 세션 조율
* **[Hooks](/hooks)** — 루프 완전 외부에서 결정론적 스크립트 실행
* **[플러그인](/plugins)** 및 **[마켓플레이스](/plugin-marketplaces)** — 기능 패키징 및 배포

[Skills](/skills)는 가장 유연한 확장입니다. 스킬은 Claude가 호출할 수 있는 마크다운 파일로서, 지식, 워크플로우, 또는 반복 가능한 작업을 포함할 수 있습니다. 스킬은 `/deploy` 같은 명령으로 호출할 수도 있고, Claude가 관련성이 있으면 자동으로 로드될 수 있습니다. 스킬은 현재 대화나 고립 컨텍스트에서, 또는 서브에이전트를 통한 고립 컨텍스트에서 실행될 수 있습니다.

## 목표에 맞는 기능 선택

기능은 Claude가 모든 세션에서 항상 보는 컨텍스트부터, 사용자나 Claude가 호출할 수 있는 주문형 기능, 특정 이벤트에서 실행되는 백그라운드 자동화까지 다양하게 만들어집니다. 아래 표는 무엇이 가능하고 각각 언제 사용해야 하는지 설명해줍니다.

| 기능 | 하는 일 | 언제 사용하는 시기 | 예시 |
| --- | --- | --- | --- |
| **[CLAUDE.md](/memory)** | 모든 대화에 로드되는 영구 컨텍스트 | 프로젝트 컨벤션, "항상 X 하기" 규칙 | "pnpm을 사용하세요, npm 말고. 테스트 전에 커밋하세요." |
| **[Skill](/skills)** | Claude가 사용할 수 있는 지식, 워크플로우, 반복 가능한 작업 | 재사용 가능한 콘텐츠, 참조 문서, 반복 작업 | `/deploy`가 배포 체크리스트를 실행; 엔드포인트 패턴이 있는 API 문서 스킬 |
| **[서브에이전트](/sub-agents)** | 고립된 실행 컨텍스트에서 요약된 결과 반환 | 컨텍스트 격리, 병렬 작업, 전문 워커 | 많은 파일을 읽고 핵심 결과만 반환하는 리서치 작업 |
| **[에이전트 팀](/agent-teams)** | 여러 독립 Claude Code 세션 조율 | 병렬 리서치, 새 기능 개발, 디버깅 | 보안, 성능, 테스트를 동시에 확인하는 리뷰어 스폰 |
| **[MCP](/mcp)** | 외부 서비스에 연결 | 외부 데이터 또는 작업 | 데이터베이스 쿼리, Slack에 게시, 브라우저 제어 |
| **[Hook](/hooks)** | 이벤트 시점에 결정론적 스크립트 실행 | 예측 가능한 자동화, LLM 미관여 | 모든 파일 편집 후 ESLint 실행 |

**[플러그인](/plugins)**은 패키징 레이어입니다. 플러그인은 스킬, 훅, 서브에이전트, MCP 서버를 하나의 설치 가능한 단위로 번들하고, 여러 리포지토리나 다른 사람에게 배포하는 데 사용할 수 있습니다. 플러그인 스킬은 네임스페이스가 지정되어 있어서(예: `/my-plugin:review`) 여러 플러그인이 공존할 수 있습니다. 동일한 설정을 여러 리포지토리에서 재사용하거나 [마켓플레이스](/plugin-marketplaces)를 통해 다른 사람에게 배포할 때 플러그인을 사용하세요.

### 시간이 지나면서 설정 구축하기

모든 것을 미리 구성할 필요는 없습니다. 각 기능에는 인식 가능한 트리거가 있으며, 대부분의 팀은 대략 다음 순서로 추가합니다:

| 트리거 | 추가할 것 |
| --- | --- |
| Claude가 컨벤션이나 명령을 두 번 틀리거나 잘못함 | [CLAUDE.md](/memory)에 추가 |
| 동일한 프롬프트를 채팅에 계속 붙여넣음 | 사용자 호출 [스킬](/skills)로 저장 |
| 동일한 플레이북이나 다단계 절차를 채팅에 매번 입력 | [스킬](/skills)로 캡처 |
| Claude가 볼 수 없는 브라우저 탭에서 데이터를 계속 복사함 | 해당 시스템을 [MCP 서버](/mcp)로 연결 |
| 사이드 작업이 대화를 채우는 출력으로 채우고 다시 참조하지 않음 | [서브에이전트](/sub-agents)를 통해 라우팅 |
| 두 번째 리포지토리가 동일한 설정이 필요함 | [플러그인](/plugins)으로 패키징 |
| Claude에게 전용 작업을 물어보고 이벤트가 발생할 때마다 다르게 반응함 | [훅](/hooks) 작성 |

같은 트리거는 Claude가 이미 알고 있는 것을 업데이트할 때 알려줍니다. 반복된 실수나 반복되는 리뷰 코멘트는 CLAUDE.md 편집이지, 별도의 스킬이나 플러그인 파일을 한 번 더 수정이 필요합니다.

### 비슷한 기능 비교

일부 기능은 비슷해 보일 수 있습니다. 구별하는 방법은 다음과 같습니다.

::: code-group
```
Skill vs 서브에이전트:
Skills와 서브에이전트는 다른 문제를 해결합니다:

- **Skills**는 재사용 가능한 콘텐츠를 가진 지식
- **서브에이전트**는 메인 대화와 완전히 분리된 고립 컨텍스트

| 측면 | Skill | 서브에이전트 |
| --- | --- | --- |
| **정체** | 재사용 가능한 지식, 호출 가능한 워크플로우 | 자체 컨텍스트를 가진 고립 컨텍스트 |
| **핵심 이점** | 컨텍스트 간 콘텐츠 공유 | 컨텍스트 격리. 작업은 별도로 진행, 요약만 반환 |
| **가장 적합한 경우** | 참조 자료, 호출 가능한 워크플로우 | 많은 파일을 읽고 병렬 작업, 전문 워커 |
```

**스킬**은 참조 또는 액션이 될 수 있습니다.** 참조 스킬은 Claude가 사용하는 모든 도구(예: API 스타일 가이드)를 정의합니다. 액션 스킬은 Claude에게 특정 작업을 지시(예: `/deploy`가 배포 워크플로우를 실행하는)합니다.

**서브에이전트를 사용하세요** — 메인 대화의 컨텍스트는 격리되지만 서브에이전트는 고립된 컨텍스트에서 자체 루프를 실행하고 결과를 반환합니다. 서브에이전트 작업은 메인 대화에 요약만 전달되며, 중간 작업이 메인 컨텍스트를 소모하지 않으므로 관리하기 편합니다.

**결합할 수 있습니다.** 서브에이전트는 특정 스킬(`skills:` 필드)을 미리 로드할 수 있습니다. 스킬은 `context: fork`를 사용해 격리된 컨텍스트에서 실행할 수 있습니다. 자세한 내용은 [Skills](/skills)를 참조하세요.

**예시:** MCP 서버가 Claude를 데이터베이스에 연결합니다. 스킬은 팀의 데이터베이스 스키마, 일반적인 쿼리 패턴, 그리고 Slack에 게시 방법을 가르칩니다. 이는 MCP가 제공하는 기능이며, 스킬은 그 기능을 효과적으로 사용하는 방법을 설명하는 지식입니다.
```

::: code-group
```
CLAUDE.md vs Skill:
둘 다 지침을 저장하지만 로드 방식과 목적이 다릅니다:

| 측면 | CLAUDE.md | Skill |
| --- | --- | --- |
| **로드** | 모든 세션, 자동 | 주문형 |
| **파일 포함 가능** | 예, `@path` 임포트 | 예, `@path` 임포트로 |
| **워크플로우 트리거** | 아니요 | 예, `/<이름>`으로 |

**CLAUDE.md에 넣으세요** — Claude가 항상 알아야 하는 것: 코딩 컨벤션, 빌드 명령, 프로젝트 구조, "절대 X 하지 마" 규칙.

**스킬에 넣으세요** — Claude가 가끔 필요한 참조 자료(API 문서, 스타일 가이드)가 있고, 그 도구를 어떻게 사용하는지 가르치는 지식입니다. 스킬에는 `/deploy` 같은 명령으로 스킬을 호출하거나, Claude가 관련성을 가질 때마다 자동으로 로드될 수 있습니다.
```

::: code-group
```
CLAUDE.md vs Rules vs Skills:
세 가지 모두 지침을 저장하지만 로드 방식과 목적이 다릅니다:

| 측면 | CLAUDE.md | `.claude/rules/` | Skill |
| --- | --- | --- | --- |
| **로드** | 모든 세션 | 모든 세션, 또는 일치하는 파일이 열릴 때 | 주문형, 호출되거나 관련 있을 때 |
| **범위** | 프로젝트 전체 | 파일 경로에 스코프 가능 | 작업별 |
| **적합한 경우** | 핵심 컨벤션과 빌드 명령 | 언어별 또는 디렉토리별 가이드라인 | 참조 자료, 반복 가능한 워크플로우 |

**CLAUDE.md를 사용하세요** — 항상 보는 영구 컨텍스트를 위한 코딩 컨벤션, 빌드 명령, 프로젝트 구조, "절대 X 하지 마" 규칙입니다. 200줄 이하로 유지하세요. 규칙이 계속 커지면 참조 자료를 `.claude/rules/` 파일로 분리하세요.

**스킬을 사용하세요** — Claude가 가끔 필요한 참조 자료를 위한 지식과 호출 가능한 워크플로우입니다. 스킬은 현재 대화나 고립 컨텍스트에서도, 또는 서브에이전트를 통한 고립 컨텍스트에서 실행될 수 있습니다.
```

::: code-group
```
서브에이전트 vs 에이전트 팀:
둘 다 작업을 병렬화하지만 아키텍처가 다릅니다:

- **서브에이전트**는 세션 내부에서 실행되며 결과를 메인 에이전트에게만 반환
- **에이전트 팀**은 서로 소통하는 여러 독립 Claude Code 세션

| 측면 | 서브에이전트 | 에이전트 팀 |
| --- | --- | --- |
| **컨텍스트** | 고립된 컨텍스트; 결과를 호출자에게 반환 | 자체 컨텍스트 윈도우; 완전히 독립적 |
| **통신** | 메인 에이전트에게만 결과 보고 | 팀원들이 서로 직접 메시지 |
| **조율** | 메인 에이전트가 모든 작업 관리 | 공유 작업 목록으로 자체 조율 |
| **가장 적합한 경우** | 결과만 중요한 집중 작업 | 경쟁 가설로 디버깅; 전문 워커 |
| **토큰 비용** | 낮음: 결과가 메인 컨텍스트에 요약됨 | 높음: 각 팀원은 별도 Claude 인스턴스 |

**서브에이전트를 사용하세요** — 컨텍스트 격리가 필요하거나 컨텍스트 윈도우가 꽉 찰 때 유용합니다. 서브에이전트 작업은 메인 대화를 소모하지 않으므로 관리하기 편합니다. 서브에이전트는 많은 파일을 읽거나 광범위한 검색을 실행할 수 있지만, 메인 대화에는 요약만 전달됩니다.

**에이전트 팀을 사용하세요** — 여러 독립 Claude Code 세션을 조율할 때 유용합니다. 질문 조사, 주장 검증, 파일 리뷰, 보안, 성능, 테스트를 동시에 확인하는 복수의 리뷰어 스폰를 스폰하여 작업의 품질을 높일 수 있습니다. 에이전트 팀은 실험적 기능이며 기본적으로 비활성화되어 있습니다. 설정과 현재 제한사항은 [에이전트 팀](/agent-teams)을 참조하세요.

**전환 시점:** 병렬 서브에이전트를 실행 중인데 메인 대화의 컨텍스트 한도가 꽉 차거나, 에이전트 팀이 자연스러운 다음 단계입니다. 실험 기능을 사용하는 경우에는 에이전트 팀이 자연스러운 다음 단계입니다.

::: info
에이전트 팀은 실험적 기능이며 기본적으로 비활성화됩니다. 설정과 현재 제한사항은 [에이전트 팀](/agent-teams)을 참조하세요.
:::

```
MCP vs Skill:
MCP는 Claude를 외부 서비스에 연결합니다. 스킬은 Claude가 그 서비스를 효과적으로 사용하는 방법에 대한 지식을 제공합니다.

| 측면 | MCP | Skill |
| --- | --- | --- |
| **정체** | 외부 서비스 연결 프로토콜 | 지식, 워크플로우, 참조 자료 |
| **제공** | 도구와 데이터 접근 | 지식, 워크플로우, 참조 자료 |
| **예시** | 데이터베이스 쿼리, Slack에 게시, 브라우저 제어 | 코드 리뷰 체크리스트, 배포 워크플로우, API 스타일 가이드 |

이들은 다른 문제를 해결하며 잘 작동합니다:

**MCP**는 Claude가 외부 시스템과 상호작용할 수 있는 능력을 부여합니다. MCP가 없으면 Claude가 데이터베이스를 쿼리하거나 Slack에 게시할 수 없습니다.

**스킬**은 Claude에게 그 도구를 효과적으로 사용하는 방법에 대한 지식을 제공합니다. MCP 서버가 Claude를 데이터베이스에 연결하면, 스킬은 팀의 데이터베이스 스키마, 일반적인 쿼리 패턴, 그리고 Slack에 게시 방법을 가르칩니다. 이는 MCP가 제공하는 기능이며, 스킬은 그 기능을 효과적으로 사용하는 방법을 설명하는 지식입니다.
```

::: code-group
```
서브에이전트 vs Hook:
둘 다 실행 외부에서 결정론적으로 동작하지만 목적이 다릅니다:

- **서브에이전트** — 루프가 자체적으로 실행됨
- **훅** — 루프 완전 외부에서 실행됨

| 측면 | 서브에이전트 | Hook |
| --- | --- | --- |
| **실행** | 격리된 컨텍스트 내부; 요약 반환 | 이벤트 시점에 결정론적 스크립트 |
| **결정론** | 예측 가능한 자동화 | LLM 미관여 |
| **가장 적합한 경우** | 부작용 | 사이드 이펙트, 알림 전송 | 편집 후 린트, 로깅, 알림 |

**서브에이전트를 사용하세요** — 컨텍스트 격리가 필요하거나 메인 대화에 채워지는 작업에 유용합니다. 서브에이전트 작업은 메인 대화를 소모하지 않으므로 관리하기 편합니다.

**훅을 사용하세요** — 이벤트 시점에 결정론적 스크립트를 실행합니다. 훅은 파일 편집, 모든 파일 편집 후 린트 실행, 로깅, 알림 전송과 같은 사이드 이펙트에 가장 적합합니다.
```

## 컨텍스트 비용 이해하기

확장을 추가하면 Claude의 컨텍스트 공간이 일부 소모됩니다. 너무 많으면 컨텍스트 윈도우가 가득 차고, 노이즈가 증가해 Claude가 덜 효과적일 수 있습니다. 기능마다 다른 로딩 전략과 비용이 있습니다.

::: info
모든 기능을 항상 보는 컨텍스트부터 사용자나 Claude가 호출할 수 있는 주문형 기능, 특정 이벤트에서 실행되는 백그라운드 자동화까지 다양하게 만들어집니다. 각 기능 추가 시 컨텍스트 비용을 고려하세요. 대화형 상호작용에 대한 대화형 뷰는 [컨텍스트 윈도우 살펴보기](/context-window)를 참조하세요.
:::

### 기능별 컨텍스트 비용

각 기능은 세션의 다른 시점에 로드됩니다:

| 기능 | 로드 시점 | 로드되는 것 | 컨텍스트 비용 |
| --- | --- | --- | --- |
| **[CLAUDE.md](/memory)** | 세션 시작 | 전체 콘텐츠 | 모든 요청 |
| **[Skills](/skills)** | 세션 시작 + 사용 시 | 설명(시작 시), 전체 콘텐츠(사용 시) | 낮음(모든 요청에 설명만)\\* |
| **[MCP 서버](/mcp)** | 세션 시작 | 도구 이름; 전체 스키마(사용 시) | 낮음(사용 시)까지 |
| **[서브에이전트](/sub-agents)** | 스폰 시 | 고립 컨텍스트; 지정 스킬, CLAUDE.md, git 상태 | 고립될 때마다 전체 공간 |
| **[Hooks](/hooks)** | 트리거 시점 | 없음(외부에서 실행) | 0, 하지만 훅이 결과를 추가하면 전달됨 |
| **[플러그인](/plugins)** | 사용 시 | 플러그인 콘텐츠 | 플러그인에 따라 다름 |

\*기본적으로 스킬 설명은 세션 시작 시 모델 인보케이션으로 로드됩니다. 사용자만 호출(`/<이름>`)하는 스킬의 경우 스킬의 전체 콘텐츠도 모델에 추가됩니다. 사용자만 스킬(`disable-model-invocation: true`) 있는 스킬은 모델 인보케이션 로드되지 않으며 0 컨텍스트 비용을 유지합니다. [자동 메모리](/memory#auto-memory)를 사용할 때는 기억하세요.
:::

### 기능 결합하기

각 확장은 다른 문제를 해결합니다. CLAUDE.md는 항상 보는 컨텍스트, 스킬은 필요할 때만 호출 가능한 지식, MCP는 외부 연결을 제공하고, 서브에이전트는 격리된 작업을 제공합니다. 실제 설정은 워크플로우에 따라 이들을 결합합니다.

예를 들어, 프로젝트 컨벤션을 위해 [CLAUDE.md](/memory)를 사용하고, 배포 워크플로우는 `/deploy` 스킬을 사용하거나, MCP를 통해 데이터베이스에 연결하여 Slack에 게시하고, 훅을 사용하여 파일 편집 후 ESLint를 실행할 수 있습니다. 각 기능이 서로 다른 문제를 해결하며 잘 작동합니다.

| 패턴 | 작동 방식 | 예시 |
| --- | --- | --- |
| **[Skill](/skills) + [MCP](/mcp)** | MCP가 연결을 제공; 스킬은 사용법 설명 | MCP 서버가 Claude를 데이터베이스에 연결합니다. 스킬은 팀의 데이터베이스 스키마와 쿼리 패턴을 가르치고 Slack 게시 방법을 포함 |
| **[Skill](/skills) + [서브에이전트](/sub-agents)** | 스킬이 서브에이전트를 스폰 | `/audit` 스킬이 보안, 성능, 스타일 서브에이전트를 스폰하여 보안, 성능, 테스트 리뷰를 실행 |
| **[CLAUDE.md](/memory) + [Skills](/skills)** | CLAUDE.md는 항상 로드; 스킬은 관련성이 있을 때 로드될 수 있음 | 스킬은 API 스타일 가이드를 포함하고 `/deploy` 같은 호출 가능한 명령을 제공합니다. |

::: info
모든 기능이 Claude의 컨텍스트 공간을 일부 소모합니다. 너무 많으면 컨텍스트 윈도우가 가득 차고, 노이즈가 증가해 Claude가 덜 효과적일 수 있습니다. 기능마다 다른 로딩 전략과 비용이 있습니다. 대화형 상호작용에 대한 대화형 뷰는 [컨텍스트 윈도우 살펴보기](/context-window)를 참조하세요.
:::
