# 컨텍스트 윈도우 탐색

Claude Code의 컨텍스트 윈도우는 세션에서 Claude가 알고 있는 모든 것을 담고 있습니다: 여러분의 지시사항, 읽은 파일, Claude 자신의 응답, 그리고 터미널에 표시되지 않는 콘텐츠가 포함됩니다. 아래의 타임라인은 무엇이 언제 로드되는지를 보여줍니다. 동일한 내용을 목록으로 보려면 [아래의 상세 설명](#타임라인이-보여주는-것)을 참조하세요.

::: info
이 페이지의 원본에는 컨텍스트 윈도우가 채워지는 과정을 시각화하는 인터랙티브 타임라인이 포함되어 있습니다. 인터랙티브 버전은 [원본 영문 문서](https://docs.anthropic.com/en/docs/claude-code/context-window)에서 확인할 수 있습니다.
:::

## 타임라인이 보여주는 것

이 세션은 대표적인 토큰 수와 함께 현실적인 흐름을 보여줍니다:

* **아무것도 입력하기 전**: CLAUDE.md, 자동 메모리, MCP 도구 이름, Skills 설명이 모두 컨텍스트에 로드됩니다. 여러분의 설정에 따라 [출력 스타일](/output-styles)이나 [`--append-system-prompt`](/cli-reference)의 텍스트 등이 추가로 로드될 수 있으며, 이들은 같은 방식으로 시스템 프롬프트에 포함됩니다.
* **Claude가 작업하는 동안**: 파일을 읽을 때마다 컨텍스트에 추가되고, [경로 기반 Rules](/memory#path-specific-rules)는 매칭되는 파일과 함께 자동으로 로드되며, [PostToolUse Hook](/hooks-guide)이 각 편집 후에 실행됩니다.
* **후속 프롬프트**: [서브에이전트](/sub-agents)가 자체 별도 컨텍스트 윈도우에서 조사를 처리하므로, 대용량 파일 읽기가 여러분의 컨텍스트에 포함되지 않습니다. 요약과 작은 메타데이터 트레일러만 돌아옵니다.
* **마지막에**: `/compact`이 대화를 구조화된 요약으로 대체합니다. 대부분의 시작 콘텐츠는 자동으로 다시 로드됩니다. 각 메커니즘에 어떤 일이 일어나는지는 아래 표를 참조하세요.

## 압축(Compaction) 후 유지되는 것

긴 세션이 압축되면, Claude Code는 대화 기록을 요약하여 컨텍스트 윈도우에 맞춥니다. 여러분의 지시사항이 어떻게 되는지는 로드 방식에 따라 달라집니다:

| 메커니즘                                    | 압축 후                                                                                      |
| :---------------------------------------- | :------------------------------------------------------------------------------------------ |
| 시스템 프롬프트와 출력 스타일                  | 변경 없음; 메시지 기록의 일부가 아님                                                              |
| 프로젝트 루트 CLAUDE.md와 범위 없는 Rules       | 디스크에서 재주입                                                                                |
| 자동 메모리                                  | 디스크에서 재주입                                                                                |
| `paths:` 프론트매터가 있는 Rules              | 매칭되는 파일이 다시 읽힐 때까지 손실                                                               |
| 하위 디렉토리의 중첩 CLAUDE.md                 | 해당 하위 디렉토리의 파일이 다시 읽힐 때까지 손실                                                     |
| 호출된 Skill 본문                             | 재주입되며, Skill당 5,000 토큰, 총 25,000 토큰으로 제한; 가장 오래된 것부터 제거                        |
| Hooks                                     | 해당 없음; Hooks는 컨텍스트가 아닌 코드로 실행됨                                                     |

경로 기반 Rules와 중첩 CLAUDE.md 파일은 트리거 파일이 읽힐 때 메시지 기록에 로드되므로, 압축 시 다른 모든 것과 함께 요약됩니다. Claude가 매칭되는 파일을 다시 읽으면 다시 로드됩니다. Rule이 압축 후에도 유지되어야 한다면, `paths:` 프론트매터를 제거하거나 프로젝트 루트 CLAUDE.md로 이동하세요.

Skill 본문은 압축 후 재주입되지만, 큰 Skill은 Skill당 제한에 맞게 잘리며, 총 예산이 초과되면 가장 오래전에 호출된 Skill부터 제거됩니다. 잘림은 파일의 시작 부분을 유지하므로, 가장 중요한 지시사항을 `SKILL.md` 상단에 배치하세요.

## 자신의 세션 확인하기

시각화에 사용된 수치는 대표적인 값입니다. 실제 컨텍스트 사용량을 확인하려면, `/context`를 실행하여 카테고리별 실시간 분석과 최적화 제안을 확인하세요. `/memory`를 실행하면 시작 시 어떤 CLAUDE.md와 자동 메모리 파일이 로드되었는지 확인할 수 있습니다.
