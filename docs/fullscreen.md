# 전체 화면 렌더링

::: info
전체 화면 렌더링은 옵트인 방식의 [리서치 프리뷰](#리서치-프리뷰)이며 Claude Code v2.1.89 이상이 필요합니다. `CLAUDE_CODE_NO_FLICKER=1`로 활성화하세요. 피드백에 따라 동작이 변경될 수 있습니다.
:::

전체 화면 렌더링은 깜빡임을 제거하고, 긴 대화에서 메모리 사용량을 일정하게 유지하며, 마우스 지원을 추가하는 Claude Code CLI의 대체 렌더링 경로입니다. `vim`이나 `htop`처럼 터미널의 대체 화면 버퍼에 인터페이스를 그리며, 현재 보이는 메시지만 렌더링합니다. 이렇게 하면 각 업데이트 시 터미널에 전송되는 데이터 양이 줄어듭니다.

이 차이는 VS Code 통합 터미널, tmux, iTerm2 등 렌더링 처리량이 병목인 터미널 에뮬레이터에서 가장 눈에 띕니다. Claude가 작업하는 동안 터미널 스크롤 위치가 상단으로 점프하거나, 도구 출력이 스트리밍될 때 화면이 깜빡이는 경우 이 모드가 이를 해결합니다.

::: info
전체 화면이라는 용어는 `vim`이 하는 것처럼 Claude Code가 터미널의 그리기 표면을 차지하는 방식을 설명합니다. 터미널 창을 최대화하는 것과는 관련이 없으며, 어떤 창 크기에서든 작동합니다.
:::

## 전체 화면 렌더링 활성화

Claude Code를 시작할 때 `CLAUDE_CODE_NO_FLICKER` 환경 변수를 설정하세요:

```bash
CLAUDE_CODE_NO_FLICKER=1 claude
```

모든 세션에서 활성화하려면 `~/.zshrc` 또는 `~/.bashrc` 등의 셸 프로필에서 변수를 내보내세요:

```bash
export CLAUDE_CODE_NO_FLICKER=1
```

## 변경 사항

전체 화면 렌더링은 CLI가 터미널에 그리는 방식을 변경합니다. 입력 상자는 출력이 스트리밍될 때 이동하지 않고 화면 하단에 고정됩니다. Claude가 작업하는 동안 입력이 제자리에 있으면 전체 화면 렌더링이 활성화된 것입니다. 보이는 메시지만 렌더 트리에 유지되므로 대화 길이에 관계없이 메모리가 일정합니다.

대화가 터미널의 스크롤백이 아닌 대체 화면 버퍼에 있기 때문에 몇 가지가 다르게 작동합니다:

| 이전                                               | 현재                                                                                                                                                              | 세부 사항                                                              |
| :------------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------- |
| `Cmd+f` 또는 tmux 검색으로 텍스트 찾기              | `Ctrl+o` 한 번으로 트랜스크립트 모드 (`/`로 검색하거나 `[`로 스크롤백에 쓰기), 또는 `Ctrl+o` 두 번으로 포커스 뷰 (마지막 프롬프트 + 도구 요약 + 응답) | [대화 검색 및 검토](#대화-검색-및-검토) |
| 터미널의 기본 클릭 앤 드래그로 선택 및 복사          | 앱 내 선택, 마우스 릴리스 시 자동 복사                                                                                                                           | [마우스 사용](#마우스-사용)                                             |
| `Cmd` 클릭으로 URL 열기                              | URL 클릭                                                                                                                                                          | [마우스 사용](#마우스-사용)                                             |

마우스 캡처가 워크플로를 방해하는 경우, 깜빡임 없는 렌더링을 유지하면서 [끌 수 있습니다](#기본-텍스트-선택-유지).

## 마우스 사용

전체 화면 렌더링은 마우스 이벤트를 캡처하여 Claude Code 내부에서 처리합니다:

* **프롬프트 입력에서 클릭**하여 입력 중인 텍스트의 아무 곳에나 커서를 위치시킵니다.
* **접힌 도구 결과를 클릭**하여 전체 출력을 펼칩니다. 다시 클릭하면 접힙니다. 도구 호출과 그 결과가 함께 펼쳐집니다. 더 표시할 내용이 있는 메시지만 클릭 가능합니다.
* **URL 또는 파일 경로를 클릭**하여 엽니다. Edit이나 Write 후 출력되는 것과 같은 도구 출력의 파일 경로는 기본 애플리케이션에서 열립니다. 일반 `http://` 및 `https://` URL은 브라우저에서 열립니다. 대부분의 터미널에서 이는 마우스 캡처가 가로채는 기본 `Cmd` 클릭 또는 `Ctrl` 클릭을 대체합니다. VS Code 통합 터미널 및 유사한 xterm.js 기반 터미널에서는 계속 `Cmd` 클릭을 사용하세요. Claude Code는 링크가 두 번 열리는 것을 방지하기 위해 해당 터미널의 자체 링크 핸들러에 양보합니다.
* **클릭 앤 드래그**로 대화의 아무 곳에서나 텍스트를 선택합니다. 더블 클릭은 단어를 선택하며, iTerm2의 단어 경계를 따라 파일 경로가 하나의 단위로 선택됩니다. 트리플 클릭은 줄을 선택합니다.
* **마우스 휠로 스크롤**하여 대화를 이동합니다.

선택한 텍스트는 마우스 릴리스 시 자동으로 클립보드에 복사됩니다. 이를 끄려면 `/config`에서 Copy on select를 토글하세요. 끄면 `Ctrl+Shift+c`를 눌러 수동으로 복사합니다. kitty, WezTerm, Ghostty, iTerm2와 같이 kitty 키보드 프로토콜을 지원하는 터미널에서는 `Cmd+c`도 작동합니다. 선택이 활성화되어 있으면 `Ctrl+c`는 취소 대신 복사합니다.

## 대화 스크롤

전체 화면 렌더링은 앱 내에서 스크롤을 처리합니다. 다음 단축키로 탐색하세요:

| 단축키            | 동작                                               |
| :---------------- | :------------------------------------------------- |
| `PgUp` / `PgDn`  | 반 화면씩 위 또는 아래로 스크롤                    |
| `Ctrl+Home`      | 대화 시작으로 이동                                  |
| `Ctrl+End`       | 최신 메시지로 이동하고 자동 따라가기 다시 활성화    |
| 마우스 휠         | 한 번에 몇 줄씩 스크롤                              |

MacBook 키보드처럼 전용 `PgUp`, `PgDn`, `Home`, `End` 키가 없는 키보드에서는 `Fn`을 화살표 키와 함께 누르세요: `Fn+↑`는 `PgUp`, `Fn+↓`는 `PgDn`, `Fn+←`는 `Home`, `Fn+→`는 `End`를 보냅니다. 따라서 `Ctrl+Fn+→`가 맨 아래로 이동하는 단축키가 됩니다. 불편하다면 마우스 휠로 맨 아래로 스크롤하여 따라가기를 재개하거나, `scroll:bottom`을 접근하기 쉬운 키로 리바인딩하세요.

위로 스크롤하면 자동 따라가기가 일시 중지되어 새 출력이 맨 아래로 끌어내리지 않습니다. `Ctrl+End`를 누르거나 맨 아래로 스크롤하여 따라가기를 재개하세요.

이러한 동작은 리바인딩 가능합니다. 기본 바인딩이 없는 반 페이지 및 전체 페이지 변형을 포함한 전체 동작 이름 목록은 [스크롤 동작](/keybindings#scroll-actions)을 참조하세요.

마우스 휠 스크롤은 터미널이 마우스 이벤트를 Claude Code에 전달해야 합니다. 대부분의 터미널은 애플리케이션이 요청하면 이를 수행합니다. iTerm2에서는 프로필별 설정입니다: 휠이 아무 것도 하지 않지만 `PgUp`과 `PgDn`은 작동하는 경우, Settings → Profiles → Terminal에서 Enable mouse reporting을 켜세요. 클릭하여 펼치기와 텍스트 선택이 작동하려면 같은 설정이 필요합니다.

### 휠 스크롤 속도 조정

마우스 휠 스크롤이 느리게 느껴진다면 터미널이 배수 없이 물리적 노치당 하나의 스크롤 이벤트를 보내는 것일 수 있습니다. Ghostty나 빠른 스크롤이 활성화된 iTerm2 같은 일부 터미널은 이미 휠 이벤트를 증폭합니다. VS Code 통합 터미널을 포함한 다른 터미널은 노치당 정확히 하나의 이벤트를 보냅니다. Claude Code는 어느 쪽인지 감지할 수 없습니다.

`CLAUDE_CODE_SCROLL_SPEED`를 설정하여 기본 스크롤 거리를 배수로 조정하세요:

```bash
export CLAUDE_CODE_SCROLL_SPEED=3
```

값 `3`은 `vim` 및 유사한 애플리케이션의 기본값과 일치합니다. 이 설정은 1에서 20 사이의 값을 허용합니다.

## 대화 검색 및 검토

전체 화면 렌더링에서 `Ctrl+o`는 세 가지 상태를 순환합니다: 일반 프롬프트, 트랜스크립트 모드, 포커스 뷰. 한 번 누르면 트랜스크립트 모드로 진입하고, 다시 누르면 마지막 프롬프트, 도구 호출의 한 줄 요약과 편집 diffstat, 최종 응답만 보여주는 포커스 뷰로 돌아갑니다. 세 번째 누르면 일반 프롬프트 화면으로 돌아갑니다.

트랜스크립트 모드에서는 `less` 스타일의 탐색과 검색이 가능합니다:

| 키                                   | 동작                                                                                                |
| :----------------------------------- | :-------------------------------------------------------------------------------------------------- |
| `/`                                  | 검색 열기. 입력하여 일치 항목 찾기, `Enter`로 수락, `Esc`로 취소하고 스크롤 위치 복원               |
| `n` / `N`                            | 다음 또는 이전 일치 항목으로 이동. 검색 바를 닫은 후에도 작동                                       |
| `j` / `k` 또는 `↑` / `↓`             | 한 줄 스크롤                                                                                         |
| `g` / `G` 또는 `Home` / `End`        | 상단 또는 하단으로 이동                                                                              |
| `Ctrl+u` / `Ctrl+d`                  | 반 페이지 스크롤                                                                                     |
| `Ctrl+b` / `Ctrl+f` 또는 `Space` / `b` | 전체 페이지 스크롤                                                                                   |
| `Ctrl+o`                             | 포커스 뷰로 전환                                                                                     |
| `Esc` 또는 `q`                       | 트랜스크립트 모드를 종료하고 프롬프트로 돌아가기                                                     |

터미널의 `Cmd+f`와 tmux 검색은 대화가 기본 스크롤백이 아닌 대체 화면 버퍼에 있기 때문에 대화를 보지 못합니다. 콘텐츠를 터미널에 돌려주려면 `Ctrl+o`를 눌러 먼저 트랜스크립트 모드로 진입한 다음:

* **`[`**: 모든 도구 출력이 펼쳐진 상태로 전체 대화를 터미널의 기본 스크롤백 버퍼에 씁니다. 이제 대화는 터미널의 일반 텍스트이므로 `Cmd+f`, tmux 복사 모드, 기타 기본 도구로 검색하거나 선택할 수 있습니다. 긴 세션은 이 작업 중 잠시 멈출 수 있습니다. 이는 `Esc` 또는 `q`로 트랜스크립트 모드를 종료할 때까지 유지되며, 전체 화면 렌더링으로 돌아갑니다. 다음 `Ctrl+o`는 새로 시작합니다.
* **`v`**: 대화를 임시 파일에 쓰고 `$VISUAL` 또는 `$EDITOR`에서 엽니다.

`Esc` 또는 `q`를 눌러 프롬프트로 돌아갑니다.

## tmux와 함께 사용

전체 화면 렌더링은 tmux 내에서 작동하지만, 두 가지 주의 사항이 있습니다.

마우스 휠 스크롤은 tmux의 마우스 모드가 필요합니다. `~/.tmux.conf`에서 아직 활성화하지 않았다면 이 줄을 추가하고 설정을 다시 로드하세요:

```bash
set -g mouse on
```

마우스 모드가 없으면 휠 이벤트가 Claude Code 대신 tmux로 전달됩니다. `PgUp`과 `PgDn`을 사용한 키보드 스크롤은 어떤 경우에도 작동합니다. Claude Code는 마우스 모드가 꺼진 tmux를 감지하면 시작 시 한 번 힌트를 출력합니다.

전체 화면 렌더링은 iTerm2의 tmux 통합 모드(`tmux -CC`로 진입하는 모드)와 호환되지 않습니다. 통합 모드에서 iTerm2는 tmux가 터미널에 그리는 대신 각 tmux 패인을 기본 분할로 렌더링합니다. 대체 화면 버퍼와 마우스 추적이 제대로 작동하지 않습니다: 마우스 휠이 아무 것도 하지 않고, 더블 클릭이 터미널 상태를 손상시킬 수 있습니다. `tmux -CC` 세션에서는 전체 화면 렌더링을 활성화하지 마세요. `-CC` 없이 iTerm2 내의 일반 tmux는 잘 작동합니다.

## 기본 텍스트 선택 유지

마우스 캡처는 특히 SSH를 통하거나 tmux 내에서 가장 흔한 마찰 포인트입니다. Claude Code가 마우스 이벤트를 캡처하면 터미널의 기본 선택 시 복사가 작동하지 않습니다. 클릭 앤 드래그로 만드는 선택은 터미널의 선택 버퍼가 아닌 Claude Code 내에 존재하므로, tmux 복사 모드, Kitty 힌트 등의 도구가 이를 보지 못합니다.

Claude Code는 선택 항목을 클립보드에 쓰려고 시도하지만, 사용하는 경로는 설정에 따라 다릅니다. tmux 내에서는 tmux 붙여넣기 버퍼에 씁니다. SSH를 통해서는 일부 터미널이 기본적으로 차단하는 OSC 52 이스케이프 시퀀스로 폴백합니다. Claude Code는 각 복사 후 어떤 경로를 사용했는지 알려주는 토스트를 출력합니다.

터미널의 기본 선택에 의존하는 경우, `CLAUDE_CODE_DISABLE_MOUSE=1`을 설정하여 깜빡임 없는 렌더링과 일정한 메모리를 유지하면서 마우스 캡처를 옵트아웃하세요:

```bash
CLAUDE_CODE_NO_FLICKER=1 CLAUDE_CODE_DISABLE_MOUSE=1 claude
```

마우스 캡처가 비활성화되면 `PgUp`, `PgDn`, `Ctrl+Home`, `Ctrl+End`를 사용한 키보드 스크롤은 여전히 작동하며, 터미널이 기본적으로 선택을 처리합니다. 클릭으로 커서 위치 지정, 클릭으로 도구 출력 펼치기, URL 클릭, Claude Code 내 휠 스크롤을 사용할 수 없게 됩니다.

## 리서치 프리뷰

전체 화면 렌더링은 리서치 프리뷰 기능입니다. 일반적인 터미널 에뮬레이터에서 테스트되었지만, 덜 일반적인 터미널이나 비정상적인 구성에서는 렌더링 문제가 발생할 수 있습니다.

문제가 발생하면 Claude Code 내에서 `/feedback`을 실행하여 보고하거나, [claude-code GitHub 저장소](https://github.com/anthropics/claude-code/issues)에서 이슈를 열어주세요. 터미널 에뮬레이터 이름과 버전을 포함하세요.

전체 화면 렌더링을 끄려면 환경 변수를 해제하거나 `CLAUDE_CODE_NO_FLICKER=0`으로 설정하세요.
