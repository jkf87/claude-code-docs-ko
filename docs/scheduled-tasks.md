# 예약 작업으로 프롬프트 실행

> `/loop`와 cron 스케줄링 도구를 사용하여 Claude Code 세션 내에서 프롬프트를 반복 실행하거나, 상태를 폴링하거나, 일회성 알림을 설정하세요.

::: info
예약 작업은 Claude Code v2.1.72 이상이 필요합니다. `claude --version`으로 현재 버전을 확인하세요.
:::

예약 작업을 사용하면 Claude가 지정된 간격으로 프롬프트를 자동으로 재실행합니다. 배포 상태 폴링, PR 모니터링, 장시간 실행 중인 빌드 확인, 또는 세션 중 나중에 할 일 알림 등에 활용할 수 있습니다. 폴링 대신 이벤트 발생 즉시 반응하려면 [Channels](/channels)를 참고하세요. CI에서 실패 정보를 세션으로 직접 푸시할 수 있습니다.

작업은 세션 범위로 동작합니다. 현재 Claude Code 프로세스 내에서만 유지되며, 세션을 종료하면 사라집니다. 재시작 후에도 유지되는 지속성 있는 스케줄링을 원한다면 [Cloud](/web-scheduled-tasks) 또는 [Desktop](/desktop-scheduled-tasks) 예약 작업을 사용하거나, [GitHub Actions](/github-actions)를 활용하세요.

## 스케줄링 옵션 비교

Claude Code는 반복 작업을 예약하는 세 가지 방법을 제공합니다.

|                            | [Cloud](/web-scheduled-tasks) | [Desktop](/desktop-scheduled-tasks) | [`/loop`](/scheduled-tasks) |
| :------------------------- | :---------------------------- | :---------------------------------- | :--------------------------- |
| 실행 위치                  | Anthropic 클라우드            | 내 머신                             | 내 머신                      |
| 머신 켜짐 필요 여부        | 불필요                        | 필요                                | 필요                         |
| 열린 세션 필요 여부        | 불필요                        | 불필요                              | 필요                         |
| 재시작 후 지속성           | 있음                          | 있음                                | 없음 (세션 범위)             |
| 로컬 파일 접근             | 없음 (새 클론)                | 있음                                | 있음                         |
| MCP 서버                   | 작업별 커넥터 구성            | [설정 파일](/mcp) 및 커넥터         | 세션에서 상속                |
| 권한 프롬프트              | 없음 (자율 실행)              | 작업별 구성 가능                    | 세션에서 상속                |
| 커스텀 스케줄              | CLI에서 `/schedule` 사용      | 있음                                | 있음                         |
| 최소 실행 간격             | 1시간                         | 1분                                 | 1분                          |

::: tip
머신 없이도 안정적으로 실행해야 하는 작업에는 **cloud tasks**를 사용하세요. 로컬 파일과 도구에 접근해야 할 때는 **Desktop tasks**를 사용하세요. 세션 중 빠른 폴링에는 **`/loop`**를 사용하세요.
:::

## /loop로 프롬프트 반복 실행

`/loop` [번들 스킬](/commands)은 세션이 열려 있는 동안 프롬프트를 반복 실행하는 가장 빠른 방법입니다. 간격과 프롬프트는 모두 선택 사항이며, 제공하는 내용에 따라 루프 동작이 달라집니다.

| 제공 내용               | 예시                        | 동작                                                                                                         |
| :---------------------- | :-------------------------- | :----------------------------------------------------------------------------------------------------------- |
| 간격 + 프롬프트         | `/loop 5m check the deploy` | 프롬프트가 [고정 스케줄](#고정-간격으로-실행)로 실행됩니다                                                  |
| 프롬프트만              | `/loop check the deploy`    | 프롬프트가 매 반복마다 [Claude가 선택한 간격](#claude가-간격-선택)으로 실행됩니다                           |
| 간격만 또는 아무것도 없음 | `/loop`                   | [기본 제공 유지보수 프롬프트](#기본-제공-유지보수-프롬프트-실행)가 실행되거나, `loop.md`가 있으면 그게 사용됩니다 |

프롬프트로 다른 명령어를 전달할 수도 있습니다. 예를 들어 `/loop 20m /review-pr 1234`와 같이 사용하면 매 반복마다 패키지된 워크플로우를 재실행합니다.

### 고정 간격으로 실행

간격을 지정하면 Claude는 이를 cron 표현식으로 변환하고, 작업을 스케줄링하며, 주기와 작업 ID를 확인해 줍니다.

```text
/loop 5m check if the deployment finished and tell me what happened
```

간격은 `30m`처럼 토큰 형태로 프롬프트 앞에 오거나, `every 2 hours`처럼 문장 형태로 뒤에 올 수 있습니다. 지원되는 단위는 초(`s`), 분(`m`), 시간(`h`), 일(`d`)입니다.

cron은 1분 단위의 세밀도를 가지므로, 초는 가장 가까운 분 단위로 올림됩니다. `7m`이나 `90m`처럼 깔끔한 cron 스텝으로 매핑되지 않는 간격은 가장 가까운 간격으로 반올림되며, Claude가 선택한 값을 알려줍니다.

### Claude가 간격 선택

간격을 생략하면 Claude는 고정 cron 스케줄 대신 동적으로 간격을 선택합니다. 각 반복 후 관찰한 내용을 바탕으로 1분에서 1시간 사이의 대기 시간을 선택합니다. 빌드가 완료되거나 PR이 활발할 때는 짧게, 대기 중인 것이 없을 때는 길게 기다립니다. 선택된 대기 시간과 그 이유가 각 반복 끝에 출력됩니다.

아래 예시는 CI와 리뷰 댓글을 확인하며, PR이 조용해지면 반복 간격이 길어집니다.

```text
/loop check whether CI passed and address any review comments
```

동적 `/loop` 스케줄을 요청하면 Claude는 [Monitor 도구](/tools-reference#monitor-tool)를 직접 사용할 수 있습니다. Monitor는 백그라운드 스크립트를 실행하고 각 출력 라인을 스트리밍으로 반환하므로, 폴링 없이도 동작하며 간격으로 프롬프트를 재실행하는 것보다 토큰 효율이 높고 더 빠르게 반응합니다.

동적으로 예약된 루프는 다른 작업과 마찬가지로 [예약 작업 목록](#예약-작업-관리)에 표시되므로, 동일한 방법으로 조회하거나 취소할 수 있습니다. [지터 규칙](#지터)은 적용되지 않지만, [7일 만료](#7일-만료)는 적용됩니다. 루프는 시작 후 7일이 지나면 자동으로 종료됩니다.

::: info
Bedrock, Vertex AI, Microsoft Foundry에서는 간격 없이 프롬프트만 사용할 경우 고정 10분 스케줄로 실행됩니다.
:::

### 기본 제공 유지보수 프롬프트 실행

프롬프트를 생략하면 Claude는 사용자가 제공한 프롬프트 대신 기본 제공 유지보수 프롬프트를 사용합니다. 각 반복마다 다음 순서로 작업합니다.

* 대화에서 완료되지 않은 작업 계속 진행
* 현재 브랜치의 PR 관리: 리뷰 댓글, 실패한 CI 실행, 머지 충돌 처리
* 대기 중인 작업이 없을 때 버그 찾기나 코드 단순화 같은 정리 작업 수행

Claude는 해당 범위 밖의 새로운 작업을 시작하지 않으며, 푸시나 삭제 같은 되돌릴 수 없는 작업은 트랜스크립트에서 이미 승인된 작업을 계속할 때만 진행합니다.

```text
/loop
```

단독 `/loop`는 이 프롬프트를 [동적으로 선택된 간격](#claude가-간격-선택)으로 실행합니다. `/loop 15m`처럼 간격을 추가하면 고정 스케줄로 실행됩니다. 기본 제공 프롬프트를 자신의 것으로 교체하려면 [loop.md로 기본 프롬프트 커스터마이즈](#loopmd로-기본-프롬프트-커스터마이즈)를 참고하세요.

::: info
Bedrock, Vertex AI, Microsoft Foundry에서는 프롬프트 없이 `/loop`를 사용하면 유지보수 루프가 시작되는 대신 사용법 메시지가 출력됩니다.
:::

### loop.md로 기본 프롬프트 커스터마이즈

`loop.md` 파일을 사용하면 기본 제공 유지보수 프롬프트를 자신의 지침으로 교체할 수 있습니다. 이 파일은 단독 `/loop`에 대한 단일 기본 프롬프트를 정의하며, 별도의 예약 작업 목록이 아닙니다. 커맨드라인에서 프롬프트를 제공하면 무시됩니다. 추가 프롬프트를 함께 예약하려면 `/loop <prompt>`를 사용하거나 [Claude에 직접 요청](#예약-작업-관리)하세요.

Claude는 두 위치에서 파일을 찾으며, 먼저 발견된 것을 사용합니다.

| 경로                | 범위                                              |
| :------------------ | :------------------------------------------------ |
| `.claude/loop.md`   | 프로젝트 수준. 두 파일이 모두 있을 때 우선 적용됩니다. |
| `~/.claude/loop.md` | 사용자 수준. 자체 파일이 없는 모든 프로젝트에 적용됩니다. |

파일은 특별한 구조 없이 일반 마크다운으로 작성합니다. `/loop` 프롬프트를 직접 입력하는 것처럼 작성하면 됩니다. 다음 예시는 릴리즈 브랜치를 안정적으로 유지합니다.

```markdown title=".claude/loop.md"
Check the `release/next` PR. If CI is red, pull the failing job log,
diagnose, and push a minimal fix. If new review comments have arrived,
address each one and resolve the thread. If everything is green and
quiet, say so in one line.
```

`loop.md`를 수정하면 다음 반복부터 적용되므로, 루프가 실행되는 동안에도 지침을 수정할 수 있습니다. 두 위치 모두에 `loop.md`가 없으면 루프는 기본 제공 유지보수 프롬프트로 돌아갑니다. 파일은 간결하게 유지하세요. 25,000바이트를 초과하는 내용은 잘립니다.

## 일회성 알림 설정

일회성 알림의 경우, `/loop` 대신 자연어로 원하는 내용을 설명하세요. Claude는 실행 후 자동으로 삭제되는 단일 실행 작업을 예약합니다.

```text
remind me at 3pm to push the release branch
```

```text
in 45 minutes, check whether the integration tests passed
```

Claude는 cron 표현식을 사용하여 실행 시간을 특정 분과 시간에 고정하고, 실행 시점을 확인해 줍니다.

## 예약 작업 관리

자연어로 Claude에게 작업 목록 조회나 취소를 요청하거나, 내부 도구를 직접 참조할 수 있습니다.

```text
what scheduled tasks do I have?
```

```text
cancel the deploy check job
```

내부적으로 Claude는 다음 도구를 사용합니다.

| 도구         | 목적                                                                                     |
| :----------- | :--------------------------------------------------------------------------------------- |
| `CronCreate` | 새 작업 예약. 5필드 cron 표현식, 실행할 프롬프트, 반복 여부 또는 단일 실행 여부를 입력받습니다. |
| `CronList`   | 모든 예약 작업의 ID, 스케줄, 프롬프트를 목록으로 표시합니다.                             |
| `CronDelete` | ID로 작업을 취소합니다.                                                                  |

각 예약 작업에는 `CronDelete`에 전달할 수 있는 8자리 ID가 있습니다. 세션에는 최대 50개의 예약 작업을 보유할 수 있습니다.

## 예약 작업의 실행 방식

스케줄러는 매 초마다 실행 예정 작업을 확인하고 낮은 우선순위로 큐에 추가합니다. 예약된 프롬프트는 Claude가 응답 중일 때가 아닌, 사용자 턴 사이에 실행됩니다. 작업이 예정된 시간에 Claude가 바쁜 경우, 현재 턴이 끝날 때까지 프롬프트가 대기합니다.

모든 시간은 로컬 타임존 기준으로 해석됩니다. `0 9 * * *` 같은 cron 표현식은 UTC가 아닌, Claude Code를 실행하는 곳의 오전 9시를 의미합니다.

### 지터

모든 세션이 동시에 API에 접근하는 것을 방지하기 위해, 스케줄러는 실행 시간에 작은 결정론적 오프셋을 추가합니다.

* 반복 작업은 주기의 최대 10%까지 늦게 실행되며, 최대 15분까지 지연됩니다. 매시간 실행되는 작업은 `:00`에서 `:06` 사이에 실행될 수 있습니다.
* 정시 또는 30분에 예약된 일회성 작업은 최대 90초 일찍 실행될 수 있습니다.

오프셋은 작업 ID에서 파생되므로, 동일한 작업은 항상 동일한 오프셋을 가집니다. 정확한 시간이 중요하다면 `:00`이나 `:30`이 아닌 분을 선택하세요. 예를 들어 `0 9 * * *` 대신 `3 9 * * *`을 사용하면 일회성 지터가 적용되지 않습니다.

### 7일 만료

반복 작업은 생성 후 7일이 지나면 자동으로 만료됩니다. 마지막으로 한 번 더 실행된 후 자동으로 삭제됩니다. 이는 잊혀진 루프가 무한정 실행되는 것을 방지합니다. 반복 작업을 더 오래 유지하려면 만료 전에 취소하고 다시 만들거나, 지속적인 스케줄링을 위해 [Cloud 예약 작업](/web-scheduled-tasks)이나 [Desktop 예약 작업](/desktop-scheduled-tasks)을 사용하세요.

## Cron 표현식 참조

`CronCreate`는 표준 5필드 cron 표현식을 사용합니다. `분 시 일 월 요일` 형식입니다. 모든 필드는 와일드카드(`*`), 단일 값(`5`), 스텝(`*/15`), 범위(`1-5`), 쉼표로 구분된 목록(`1,15,30`)을 지원합니다.

| 예시           | 의미                    |
| :------------- | :---------------------- |
| `*/5 * * * *`  | 5분마다                 |
| `0 * * * *`    | 매시 정각               |
| `7 * * * *`    | 매시 7분                |
| `0 9 * * *`    | 매일 오전 9시 (로컬)    |
| `0 9 * * 1-5`  | 평일 오전 9시 (로컬)    |
| `30 14 15 3 *` | 3월 15일 오후 2시 30분 (로컬) |

요일은 일요일이 `0` 또는 `7`, 토요일이 `6`입니다. `L`, `W`, `?` 같은 확장 문법과 `MON`, `JAN` 같은 이름 별칭은 지원되지 않습니다.

일(day-of-month)과 요일(day-of-week) 모두 제약이 있는 경우, 두 필드 중 하나라도 일치하면 날짜가 일치합니다. 이는 표준 vixie-cron 의미론을 따릅니다.

## 예약 작업 비활성화

환경 변수에 `CLAUDE_CODE_DISABLE_CRON=1`을 설정하면 스케줄러를 완전히 비활성화할 수 있습니다. cron 도구와 `/loop`를 사용할 수 없게 되며, 이미 예약된 작업도 실행이 중단됩니다. 비활성화 플래그의 전체 목록은 [환경 변수](/env-vars)를 참고하세요.

## 제한 사항

세션 범위 스케줄링에는 다음과 같은 본질적인 제약이 있습니다.

* Claude Code가 실행 중이고 유휴 상태일 때만 작업이 실행됩니다. 터미널을 닫거나 세션이 종료되면 모든 작업이 취소됩니다.
* 누락된 실행에 대한 캐치업이 없습니다. Claude가 장시간 요청을 처리하는 동안 예약 시간이 지나면, 누락된 간격만큼 반복하는 것이 아니라 Claude가 유휴 상태가 될 때 한 번만 실행됩니다.
* 재시작 후 지속성이 없습니다. Claude Code를 재시작하면 세션 범위의 모든 작업이 초기화됩니다.

무인으로 실행해야 하는 cron 기반 자동화가 필요하다면 다음 옵션을 고려하세요.

* [Cloud 예약 작업](/web-scheduled-tasks): Anthropic 관리 인프라에서 실행
* [GitHub Actions](/github-actions): CI에서 `schedule` 트리거 사용
* [Desktop 예약 작업](/desktop-scheduled-tasks): 로컬 머신에서 실행
