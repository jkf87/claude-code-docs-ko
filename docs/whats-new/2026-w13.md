# Week 13 · 2026년 3월 23–27일

릴리스: v2.1.83 → v2.1.85, 기능 6종

## Feature 1: Auto mode (research preview)

Auto mode는 권한 요청을 분류기에 넘기는 기능. 안전한 수정과 명령은 사용자를 방해하지 않고 실행되며, 파괴적이거나 수상한 작업은 차단되고 별도로 표시됨. 모든 파일 쓰기마다 일일이 승인하는 방식과 `--dangerously-skip-permissions`로 실행하는 방식 사이의 중간 지점.

Shift+Tab으로 auto로 전환하거나, `.claude/settings.json`의 permissions에서 `defaultMode: "auto"`로 설정 가능.

## Feature 2: Computer use (Desktop)

이제 Claude는 Claude Code Desktop 앱에서 실제 데스크톱을 제어 가능. 네이티브 앱 열기, iOS 시뮬레이터 클릭 조작, 하드웨어 제어 패널 다루기, 화면에서 변경 사항 확인까지 지원. 기본값은 비활성화이며, 각 작업 전에 확인 요청 제공.

## Feature 3: PR auto-fix (Web)

PR을 열 때 스위치만 켜 두면 됨. Claude가 CI를 지켜보다가 실패를 수정하고, 사소한 지적까지 처리한 뒤, 초록불이 켜질 때까지 푸시 진행.

## Feature 4: Transcript search (v2.1.83)

transcript 모드에서 `/`를 눌러 대화 검색 가능. `n`과 `N`으로 검색 결과 사이 이동 가능.

## Feature 5: PowerShell tool (preview, v2.1.84)

Windows에서 Bash와 함께 네이티브 PowerShell tool 지원 추가. `CLAUDE_CODE_USE_POWERSHELL_TOOL=1` 설정으로 활성화.

## Feature 6: Conditional hooks (v2.1.85)

이제 hooks에서 권한 규칙 문법을 사용하는 `"if"` 필드 선언 가능.

```json
"if": "Bash(git commit *)"
```

## 기타 개선 사항

- Plugin userConfig 공개 전환
- 붙여넣은 이미지에 `[Image #N]` 칩 삽입
- `managed-settings.d/` drop-in 디렉터리 지원
- `CwdChanged`, `FileChanged` hook 이벤트 추가
- Agents frontmatter에서 `initialPrompt` 선언 가능
- `Ctrl+X Ctrl+E`로 외부 편집기 열기
- Claude 응답 중에도 `/status` 동작
- Deep link를 선호 터미널에서 열기
- 75분 이상 유휴 상태 후 `/clear` 유도 표시
- VS Code에 rate limit 배너, Esc 두 번으로 rewind picker 지원
