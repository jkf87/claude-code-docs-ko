# 인터랙티브 모드

## 키보드 단축키

::: info 참고
키보드 단축키는 플랫폼과 터미널에 따라 다를 수 있습니다. `?`를 눌러 현재 환경에서 사용 가능한 단축키를 확인하세요.

**macOS 사용자**: Option/Alt 키 단축키(`Alt+B`, `Alt+F`, `Alt+Y`, `Alt+M`, `Alt+P`, `Alt+T`)는 터미널에서 Option을 Meta로 설정해야 합니다:

* **iTerm2**: 설정 → Profiles → Keys → Left/Right Option key를 "Esc+"로 설정
* **Terminal.app**: 설정 → Profiles → Keyboard → "Use Option as Meta Key" 체크
* **VS Code**: VS Code 설정에서 `"terminal.integrated.macOptionIsMeta": true` 설정

자세한 내용은 [터미널 설정](/terminal-config)을 참조하세요.
:::

| :------------------------------------------------ | :------------------------------------------------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |

| `Ctrl+G` 또는 `Ctrl+X Ctrl+E` | 기본 텍스트 편집기에서 열기 | 기본 텍스트 편집기에서 프롬프트 또는 커스텀 응답을 편집합니다. `Ctrl+X Ctrl+E`는 readline 네이티브 바인딩입니다 |
| `Ctrl+L` | 프롬프트 입력 지우기 | 입력한 텍스트를 지우고 대화 기록은 유지합니다 |
| `Ctrl+O` | 트랜스크립트 뷰어 토글 | 상세한 도구 사용 및 실행을 표시합니다. 기본적으로 "Called slack 3 times"처럼 한 줄로 축약되는 MCP 호출도 확장합니다. [전체 화면 렌더링](/fullscreen)에서는 일반 프롬프트, 트랜스크립트 모드, 포커스 뷰(마지막 프롬프트 + 도구 요약 + 응답) 세 가지 상태를 순환합니다 |
| `Ctrl+R` | 역방향 명령 기록 검색 | 이전 명령을 인터랙티브하게 검색합니다 |
| `Ctrl+V` 또는 `Cmd+V` (iTerm2) 또는 `Alt+V` (Windows) | 클립보드에서 이미지 붙여넣기 | 커서 위치에 `[Image #N]` 칩을 삽입하여 프롬프트에서 위치적으로 참조할 수 있습니다 |
| `Ctrl+B` | 실행 중인 작업을 백그라운드로 | bash 명령과 에이전트를 백그라운드로 전환합니다. Tmux 사용자는 두 번 누르세요 |
| `Ctrl+T` | 작업 목록 토글 | 터미널 상태 영역에서 [작업 목록](#작업-목록)을 표시하거나 숨깁니다 |
| `Left/Right 화살표` | 대화 상자 탭 순환 | 권한 대화 상자와 메뉴에서 탭 간 이동 |
| `Up/Down 화살표` | 명령 기록 탐색 | 이전 입력을 불러옵니다 |
| `Esc` + `Esc` | 되감기 또는 요약 | 코드 및/또는 대화를 이전 지점으로 복원하거나 선택한 메시지부터 요약합니다 |
| `Shift+Tab` 또는 `Alt+M` (일부 설정) | 권한 모드 순환 | `default`, `acceptEdits`, `plan` 및 `auto` 또는 `bypassPermissions` 같이 활성화한 모드를 순환합니다. [권한 모드](/permission-modes)를 참조하세요. |
| `Option+P` (macOS) 또는 `Alt+P` (Windows/Linux) | 모델 전환 | 프롬프트를 지우지 않고 모델을 전환합니다 |
| `Option+T` (macOS) 또는 `Alt+T` (Windows/Linux) | 확장 사고 토글 | 확장 사고 모드를 활성화하거나 비활성화합니다. macOS에서는 이 단축키가 작동하도록 터미널에서 Option을 Meta로 전송하도록 설정하세요 |
| `Option+O` (macOS) 또는 `Alt+O` (Windows/Linux) | 빠른 모드 토글 | [빠른 모드](/fast-mode)를 활성화하거나 비활성화합니다 |

## 명령

Claude Code에서 `/`를 입력하면 사용 가능한 모든 명령을 볼 수 있고, `/` 다음에 아무 글자나 입력하면 필터링할 수 있습니다. `/` 메뉴는 호출할 수 있는 모든 것을 표시합니다: 내장 명령, 번들 및 사용자 작성 [스킬](/skills), [플러그인](/plugins) 및 [MCP 서버](/mcp#use-mcp-prompts-as-commands)가 제공하는 명령. 일부 내장 명령은 플랫폼이나 플랜에 따라 모든 사용자에게 표시되지 않을 수 있습니다.

Claude Code에 포함된 전체 명령 목록은 [명령 참조](/commands)를 확인하세요.

## 명령 기록

Claude Code는 현재 세션의 명령 기록을 유지합니다:

* 입력 기록은 작업 디렉토리별로 저장됩니다
* `/clear`를 실행하여 새 세션을 시작하면 입력 기록이 초기화됩니다. 이전 세션의 대화는 보존되며 다시 시작할 수 있습니다.
* Up/Down 화살표로 탐색합니다 (위의 키보드 단축키 참조)
* **참고**: 기록 확장(`!`)은 기본적으로 비활성화되어 있습니다

### Ctrl+R로 역방향 검색

`Ctrl+R`을 눌러 명령 기록을 인터랙티브하게 검색합니다:

1. **검색 시작**: `Ctrl+R`을 눌러 역방향 기록 검색을 활성화합니다
2. **쿼리 입력**: 이전 명령에서 검색할 텍스트를 입력합니다. 검색어가 일치하는 결과에서 강조 표시됩니다
3. **일치 항목 탐색**: `Ctrl+R`을 다시 눌러 이전 일치 항목을 순환합니다
4. **일치 항목 수락**:
   * `Tab` 또는 `Esc`를 눌러 현재 일치 항목을 수락하고 계속 편집합니다
   * `Enter`를 눌러 수락하고 즉시 명령을 실행합니다
5. **검색 취소**:
   * `Ctrl+C`를 눌러 취소하고 원래 입력을 복원합니다
   * 빈 검색에서 `Backspace`를 눌러 취소합니다

검색은 검색어가 강조 표시된 일치하는 명령을 표시하여 이전 입력을 찾고 재사용할 수 있게 합니다.

## 백그라운드 bash 명령

Claude Code는 백그라운드에서 bash 명령 실행을 지원하여, 장시간 실행되는 프로세스가 실행되는 동안 계속 작업할 수 있습니다.

### 백그라운드 작동 방식

Claude Code가 백그라운드에서 명령을 실행하면, 비동기적으로 명령을 실행하고 즉시 백그라운드 작업 ID를 반환합니다. 명령이 백그라운드에서 계속 실행되는 동안 Claude Code는 새 프롬프트에 응답할 수 있습니다.

백그라운드에서 명령을 실행하려면 다음 중 하나를 수행할 수 있습니다:

* Claude Code에 명령을 백그라운드에서 실행하도록 프롬프트합니다
* Ctrl+B를 눌러 일반 Bash 도구 호출을 백그라운드로 이동합니다. (Tmux 사용자는 tmux의 프리픽스 키 때문에 Ctrl+B를 두 번 눌러야 합니다.)

**주요 기능:**

* 출력은 파일에 기록되며 Claude가 Read 도구를 사용하여 검색할 수 있습니다
* 백그라운드 작업에는 추적 및 출력 검색을 위한 고유 ID가 있습니다
* Claude Code가 종료되면 백그라운드 작업이 자동으로 정리됩니다
* 출력이 5GB를 초과하면 백그라운드 작업이 자동으로 종료되며, 이유를 설명하는 메모가 stderr에 기록됩니다

모든 백그라운드 작업 기능을 비활성화하려면 `CLAUDE_CODE_DISABLE_BACKGROUND_TASKS` 환경 변수를 `1`로 설정하세요. 자세한 내용은 [환경 변수](/env-vars)를 참조하세요.

**일반적으로 백그라운드로 실행하는 명령:**

* 빌드 도구 (webpack, vite, make)
* 패키지 관리자 (npm, yarn, pnpm)
* 테스트 러너 (jest, pytest)
* 개발 서버
* 장시간 실행 프로세스 (docker, terraform)

### `!` 접두사를 사용한 Bash 모드

입력 앞에 `!`를 붙여 Claude를 거치지 않고 직접 bash 명령을 실행합니다:

```bash  theme={null}
! npm test
! git status
! ls -la
```

Bash 모드:

* 명령과 출력을 대화 컨텍스트에 추가합니다
* 실시간 진행 상황과 출력을 표시합니다
* 장시간 실행 명령에 동일한 `Ctrl+B` 백그라운드 전환을 지원합니다
* Claude가 명령을 해석하거나 승인할 필요가 없습니다
* 기록 기반 자동 완성을 지원합니다: 부분 명령을 입력하고 **Tab**을 눌러 현재 프로젝트의 이전 `!` 명령에서 완성합니다
* 빈 프롬프트에서 `Escape`, `Backspace` 또는 `Ctrl+U`로 종료합니다
* `!`로 시작하는 텍스트를 빈 프롬프트에 붙여넣으면 자동으로 bash 모드로 진입하여, 입력한 `!` 동작과 일치합니다

대화 컨텍스트를 유지하면서 빠른 셸 작업에 유용합니다.

## 프롬프트 제안

세션을 처음 열면, 시작을 돕기 위해 프롬프트 입력에 회색으로 된 예시 명령이 나타납니다. Claude Code는 프로젝트의 git 기록에서 이를 선택하여, 최근 작업한 파일을 반영합니다.

Claude가 응답한 후에도, 여러 부분 요청의 후속 단계나 워크플로의 자연스러운 연속 같은 대화 기록에 기반한 제안이 계속 나타납니다.

* **Tab** 또는 **오른쪽 화살표**를 눌러 제안을 수락하거나, **Enter**를 눌러 수락하고 제출합니다
* 입력을 시작하면 제안이 사라집니다

제안은 상위 대화의 프롬프트 캐시를 재사용하는 백그라운드 요청으로 실행되므로, 추가 비용이 최소화됩니다. Claude Code는 불필요한 비용을 피하기 위해 캐시가 콜드 상태일 때 제안 생성을 건너뜁니다.

대화의 첫 번째 턴 후, 비인터랙티브 모드, plan 모드에서는 제안이 자동으로 건너뛰어집니다.

프롬프트 제안을 완전히 비활성화하려면 환경 변수를 설정하거나 `/config`에서 설정을 토글하세요:

```bash  theme={null}
export CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION=false
```

## /btw로 사이드 질문

`/btw`를 사용하여 대화 기록에 추가하지 않고 현재 작업에 대해 빠른 질문을 할 수 있습니다. 빠른 답변이 필요하지만 메인 컨텍스트를 어지럽히거나 Claude가 장시간 실행 중인 작업에서 벗어나는 것을 원하지 않을 때 유용합니다.

```
/btw what was the name of that config file again?
```

사이드 질문은 현재 대화에 대한 전체 가시성을 가지므로, Claude가 이미 읽은 코드, 이전에 내린 결정, 또는 세션의 다른 모든 것에 대해 질문할 수 있습니다. 질문과 답변은 일시적입니다: 닫을 수 있는 오버레이에 나타나며 대화 기록에 절대 들어가지 않습니다.

* **Claude가 작업 중일 때 사용 가능**: Claude가 응답을 처리하는 동안에도 `/btw`를 실행할 수 있습니다. 사이드 질문은 독립적으로 실행되며 메인 턴을 중단하지 않습니다.
* **도구 접근 없음**: 사이드 질문은 이미 컨텍스트에 있는 것에서만 답변합니다. Claude는 사이드 질문에 답변할 때 파일을 읽거나, 명령을 실행하거나, 검색할 수 없습니다.
* **단일 응답**: 후속 턴이 없습니다. 대화가 필요하면 일반 프롬프트를 대신 사용하세요.
* **낮은 비용**: 사이드 질문은 상위 대화의 프롬프트 캐시를 재사용하므로, 추가 비용이 최소화됩니다.

**Space**, **Enter** 또는 **Escape**를 눌러 답변을 닫고 프롬프트로 돌아갑니다.

`/btw`는 [서브에이전트](/sub-agents)의 반대입니다: 전체 대화를 볼 수 있지만 도구가 없는 반면, 서브에이전트는 전체 도구를 가지지만 빈 컨텍스트로 시작합니다. 이 세션에서 Claude가 이미 알고 있는 것에 대해 질문하려면 `/btw`를 사용하고, 새로운 것을 찾으려면 서브에이전트를 사용하세요.

## 작업 목록

복잡한 여러 단계의 작업을 수행할 때, Claude는 진행 상황을 추적하기 위한 작업 목록을 생성합니다. 작업은 대기 중, 진행 중, 완료를 나타내는 표시기와 함께 터미널의 상태 영역에 나타납니다.

* `Ctrl+T`를 눌러 작업 목록 뷰를 토글합니다. 디스플레이는 한 번에 최대 10개의 작업을 표시합니다
* 모든 작업을 보거나 지우려면 Claude에게 직접 요청하세요: "show me all tasks" 또는 "clear all tasks"
* 작업은 컨텍스트 압축 후에도 유지되어, 더 큰 프로젝트에서 Claude가 체계적으로 유지되도록 돕습니다
* 세션 간에 작업 목록을 공유하려면 `CLAUDE_CODE_TASK_LIST_ID`를 설정하여 `~/.claude/tasks/`에 명명된 디렉토리를 사용합니다: `CLAUDE_CODE_TASK_LIST_ID=my-project claude`

## PR 리뷰 상태

열린 풀 리퀘스트가 있는 브랜치에서 작업할 때, Claude Code는 푸터에 클릭 가능한 PR 링크(예: "PR #446")를 표시합니다. 링크에는 리뷰 상태를 나타내는 색상 밑줄이 있습니다:

* 녹색: 승인됨
* 노란색: 리뷰 대기 중
* 빨간색: 변경 요청됨
* 회색: 초안
* 보라색: 병합됨

`Cmd+클릭` (Mac) 또는 `Ctrl+클릭` (Windows/Linux)으로 링크를 클릭하여 브라우저에서 풀 리퀘스트를 엽니다. 상태는 60초마다 자동으로 업데이트됩니다.

::: info 참고
PR 상태를 사용하려면 `gh` CLI가 설치되고 인증(`gh auth login`)되어 있어야 합니다.
:::
