채널로 실행 중인 세션에 이벤트 푸시하기 ​

INFO

채널은 리서치 프리뷰 단계이며 Claude Code v2.1.80 이상이 필요합니다. claude.ai 로그인이 필요하며, Console 및 API 키 인증은 지원되지 않습니다. Team 및 Enterprise 조직은 명시적으로 활성화해야 합니다.

채널은 실행 중인 Claude Code 세션에 이벤트를 푸시하는 MCP 서버로, 터미널을 보고 있지 않을 때 발생하는 일에 Claude가 반응할 수 있게 합니다. 채널은 양방향이 가능합니다: Claude가 이벤트를 읽고 같은 채널을 통해 응답하는 채팅 브릿지처럼 작동합니다. 이벤트는 세션이 열려 있는 동안에만 도착하므로, 항상 켜진 상태로 사용하려면 백그라운드 프로세스나 지속적인 터미널에서 Claude를 실행하세요.

새 클라우드 세션을 생성하거나 폴링을 기다리는 통합과 달리, 이벤트는 이미 열려 있는 세션에 바로 도착합니다: 채널 비교를 참조하세요.

채널은 플러그인으로 설치하고 자체 자격 증명으로 구성합니다. 리서치 프리뷰에는 Telegram, Discord, iMessage가 포함되어 있습니다.

Claude가 채널을 통해 응답하면, 터미널에서는 수신 메시지는 보이지만 응답 텍스트는 보이지 않습니다. 터미널에는 도구 호출과 확인("sent" 같은)이 표시되고, 실제 응답은 다른 플랫폼에 나타납니다.

이 페이지에서 다루는 내용:

  • 지원되는 채널: Telegram, Discord, iMessage 설정
  • 채널 설치 및 실행: localhost 데모인 fakechat으로 시작
  • 메시지를 보낼 수 있는 사람: 발신자 허용 목록과 페어링 방법
  • 조직에서 채널 활성화: Team 및 Enterprise
  • 채널 비교: 웹 세션, Slack, MCP, Remote Control과의 비교

직접 채널을 만들려면 채널 레퍼런스를 참조하세요.

지원되는 채널 ​

지원되는 각 채널은 Bun이 필요한 플러그인입니다. 실제 플랫폼을 연결하기 전에 플러그인 흐름을 직접 체험하려면 fakechat 빠른 시작을 시도해 보세요.

Telegram ​

전체 Telegram 플러그인 소스를 확인하세요.

  1. Telegram 봇 생성

    Telegram에서 BotFather를 열고 /newbot을 보냅니다. 표시 이름과 bot으로 끝나는 고유한 사용자 이름을 지정하세요. BotFather가 반환하는 토큰을 복사합니다.

  2. 플러그인 설치

    Claude Code에서 다음을 실행합니다:

    /plugin install telegram@claude-plugins-official

    Claude Code가 마켓플레이스에서 플러그인을 찾을 수 없다고 보고하면, 마켓플레이스가 없거나 오래된 것입니다. /plugin marketplace update claude-plugins-official로 갱신하거나, 이전에 추가한 적이 없다면 /plugin marketplace add anthropics/claude-plugins-official을 먼저 실행하세요. 그런 다음 설치를 다시 시도합니다.

    설치 후 /reload-plugins를 실행하여 플러그인의 configure 명령을 활성화하세요.

  3. 토큰 구성

    BotFather에서 받은 토큰으로 configure 명령을 실행합니다:

    /telegram:configure <token>

    이 토큰은 ~/.claude/channels/telegram/.env에 저장됩니다. Claude Code를 시작하기 전에 셸 환경에서 TELEGRAM_BOT_TOKEN을 설정할 수도 있습니다.

  4. 채널을 활성화하고 재시작

    Claude Code를 종료하고 채널 플래그와 함께 재시작합니다. 이렇게 하면 Telegram 플러그인이 시작되어 봇의 메시지를 폴링하기 시작합니다:

    bash
    claude --channels plugin:telegram@claude-plugins-official

  5. 계정 페어링

    Telegram을 열고 봇에 아무 메시지나 보냅니다. 봇이 페어링 코드를 응답합니다.

    INFO

    봇이 응답하지 않으면, 이전 단계에서 --channels로 Claude Code를 실행 중인지 확인하세요. 봇은 채널이 활성화된 동안에만 응답할 수 있습니다.

    Claude Code에서 다음을 실행합니다:

    /telegram:access pair <code>

    그런 다음 자신의 계정만 메시지를 보낼 수 있도록 접근을 제한합니다:

    /telegram:access policy allowlist

Discord ​

전체 Discord 플러그인 소스를 확인하세요.

  1. Discord 봇 생성

    Discord Developer Portal로 이동하여 New Application을 클릭하고 이름을 지정합니다. Bot 섹션에서 사용자 이름을 만든 다음 Reset Token을 클릭하고 토큰을 복사합니다.

  2. Message Content Intent 활성화

    봇 설정에서 Privileged Gateway Intents로 스크롤하고 Message Content Intent를 활성화합니다.

  3. 봇을 서버에 초대

    OAuth2 > URL Generator로 이동합니다. bot 스코프를 선택하고 다음 권한을 활성화합니다:

    • View Channels
    • Send Messages
    • Send Messages in Threads
    • Read Message History
    • Attach Files
    • Add Reactions

    생성된 URL을 열어 봇을 서버에 추가합니다.

  4. 플러그인 설치

    Claude Code에서 다음을 실행합니다:

    /plugin install discord@claude-plugins-official

    Claude Code가 마켓플레이스에서 플러그인을 찾을 수 없다고 보고하면, 마켓플레이스가 없거나 오래된 것입니다. /plugin marketplace update claude-plugins-official로 갱신하거나, 이전에 추가한 적이 없다면 /plugin marketplace add anthropics/claude-plugins-official을 먼저 실행하세요. 그런 다음 설치를 다시 시도합니다.

    설치 후 /reload-plugins를 실행하여 플러그인의 configure 명령을 활성화하세요.

  5. 토큰 구성

    복사한 봇 토큰으로 configure 명령을 실행합니다:

    /discord:configure <token>

    이 토큰은 ~/.claude/channels/discord/.env에 저장됩니다. Claude Code를 시작하기 전에 셸 환경에서 DISCORD_BOT_TOKEN을 설정할 수도 있습니다.

  6. 채널을 활성화하고 재시작

    Claude Code를 종료하고 채널 플래그와 함께 재시작합니다. 이렇게 하면 Discord 플러그인이 연결되어 봇이 메시지를 수신하고 응답할 수 있습니다:

    bash
    claude --channels plugin:discord@claude-plugins-official

  7. 계정 페어링

    Discord에서 봇에 DM을 보냅니다. 봇이 페어링 코드를 응답합니다.

    INFO

    봇이 응답하지 않으면, 이전 단계에서 --channels로 Claude Code를 실행 중인지 확인하세요. 봇은 채널이 활성화된 동안에만 응답할 수 있습니다.

    Claude Code에서 다음을 실행합니다:

    /discord:access pair <code>

    그런 다음 자신의 계정만 메시지를 보낼 수 있도록 접근을 제한합니다:

    /discord:access policy allowlist

iMessage ​

전체 iMessage 플러그인 소스를 확인하세요.

iMessage 채널은 Messages 데이터베이스를 직접 읽고 AppleScript를 통해 응답을 보냅니다. macOS가 필요하며 봇 토큰이나 외부 서비스가 필요 없습니다.

  1. 전체 디스크 접근 권한 부여

    ~/Library/Messages/chat.db에 있는 Messages 데이터베이스는 macOS에 의해 보호됩니다. 서버가 처음 읽을 때 macOS가 접근 권한을 요청합니다: 허용을 클릭하세요. 프롬프트에는 Bun을 실행한 앱 이름(Terminal, iTerm 또는 IDE)이 표시됩니다.

    프롬프트가 나타나지 않거나 허용 안 함을 클릭한 경우, 시스템 설정 > 개인정보 보호 및 보안 > 전체 디스크 접근에서 수동으로 접근 권한을 부여하고 터미널을 추가하세요. 이 설정이 없으면 서버가 authorization denied와 함께 즉시 종료됩니다.

  2. 플러그인 설치

    Claude Code에서 다음을 실행합니다:

    /plugin install imessage@claude-plugins-official

    Claude Code가 마켓플레이스에서 플러그인을 찾을 수 없다고 보고하면, 마켓플레이스가 없거나 오래된 것입니다. /plugin marketplace update claude-plugins-official로 갱신하거나, 이전에 추가한 적이 없다면 /plugin marketplace add anthropics/claude-plugins-official을 먼저 실행하세요. 그런 다음 설치를 다시 시도합니다.

  3. 채널을 활성화하고 재시작

    Claude Code를 종료하고 채널 플래그와 함께 재시작합니다:

    bash
    claude --channels plugin:imessage@claude-plugins-official

  4. 자신에게 문자 보내기

    Apple ID로 로그인한 기기에서 Messages를 열고 자신에게 메시지를 보냅니다. 메시지가 Claude에 즉시 전달됩니다: 셀프 채팅은 별도 설정 없이 접근 제어를 우회합니다.

    INFO

    Claude가 보내는 첫 번째 응답은 터미널이 Messages를 제어할 수 있는지 묻는 macOS 자동화 프롬프트를 발생시킵니다. 확인을 클릭하세요.

  5. 다른 발신자 허용

    기본적으로 자신의 메시지만 통과합니다. 다른 연락처가 Claude에 접근할 수 있게 하려면 핸들을 추가합니다:

    /imessage:access allow +15551234567

    핸들은 +국가코드 형식의 전화번호이거나 user@example.com 같은 Apple ID 이메일입니다.

아직 플러그인이 없는 시스템을 위해 직접 채널을 만들 수도 있습니다.

빠른 시작 ​

Fakechat은 공식 지원 데모 채널로, 인증할 것도 없고 구성할 외부 서비스도 없이 localhost에서 채팅 UI를 실행합니다.

fakechat을 설치하고 활성화하면, 브라우저에서 입력한 메시지가 Claude Code 세션에 도착합니다. Claude가 응답하면, 응답이 브라우저에 다시 나타납니다. fakechat 인터페이스를 테스트한 후, Telegram, Discord, 또는 iMessage를 시도해 보세요.

fakechat 데모를 사용하려면 다음이 필요합니다:

  • claude.ai 계정으로 설치 및 인증된 Claude Code
  • Bun 설치. 사전 빌드된 채널 플러그인은 Bun 스크립트입니다. bun --version으로 확인하고, 실패하면 Bun을 설치하세요.
  • Team/Enterprise 사용자: 조직 관리자가 관리 설정에서 채널을 활성화해야 합니다

  1. fakechat 채널 플러그인 설치

    Claude Code 세션을 시작하고 설치 명령을 실행합니다:

    text
    /plugin install fakechat@claude-plugins-official

    Claude Code가 마켓플레이스에서 플러그인을 찾을 수 없다고 보고하면, 마켓플레이스가 없거나 오래된 것입니다. /plugin marketplace update claude-plugins-official로 갱신하거나, 이전에 추가한 적이 없다면 /plugin marketplace add anthropics/claude-plugins-official을 먼저 실행하세요. 그런 다음 설치를 다시 시도합니다.

  2. 채널을 활성화하고 재시작

    Claude Code를 종료한 다음 --channels와 설치한 fakechat 플러그인을 지정하여 재시작합니다:

    bash
    claude --channels plugin:fakechat@claude-plugins-official

    fakechat 서버가 자동으로 시작됩니다.

    TIP

    --channels에 여러 플러그인을 공백으로 구분하여 전달할 수 있습니다.

  3. 메시지 푸시

    http://localhost:8787에서 fakechat UI를 열고 메시지를 입력합니다:

    text
    hey, what's in my working directory?

    메시지가 <channel source="fakechat"> 이벤트로 Claude Code 세션에 도착합니다. Claude가 메시지를 읽고 작업을 수행한 다음 fakechat의 reply 도구를 호출합니다. 답변이 채팅 UI에 표시됩니다.

자리를 비운 동안 Claude가 권한 프롬프트에 부딪히면, 응답할 때까지 세션이 일시 중지됩니다. 권한 릴레이 기능을 선언한 채널 서버는 이러한 프롬프트를 전달하여 원격으로 승인하거나 거부할 수 있습니다. 무인 사용의 경우, --dangerously-skip-permissions가 프롬프트를 완전히 우회하지만, 신뢰할 수 있는 환경에서만 사용하세요.

보안 ​

승인된 모든 채널 플러그인은 발신자 허용 목록을 유지합니다: 추가한 ID만 메시지를 푸시할 수 있으며, 나머지는 조용히 차단됩니다.

Telegram과 Discord는 페어링으로 목록을 구성합니다:

  1. Telegram이나 Discord에서 봇을 찾아 아무 메시지나 보냅니다
  2. 봇이 페어링 코드를 응답합니다
  3. Claude Code 세션에서 코드 승인 프롬프트가 나타나면 승인합니다
  4. 발신자 ID가 허용 목록에 추가됩니다

iMessage는 다르게 작동합니다: 자신에게 문자를 보내면 자동으로 게이트를 우회하며, 다른 연락처는 /imessage:access allow로 핸들을 추가합니다.

이 외에도 --channels로 각 세션에서 어떤 서버를 활성화할지 제어하며, Team 및 Enterprise 플랜에서는 조직이 channelsEnabled로 가용성을 제어합니다.

.mcp.json에 있다고 해서 메시지를 푸시할 수 있는 것은 아닙니다: 서버는 --channels에도 지정되어야 합니다.

허용 목록은 채널이 선언한 경우 권한 릴레이도 제어합니다. 채널을 통해 응답할 수 있는 사람은 세션에서 도구 사용을 승인하거나 거부할 수 있으므로, 그 권한을 신뢰할 수 있는 발신자만 허용 목록에 추가하세요.

Enterprise 제어 ​

Team 및 Enterprise 플랜에서 채널은 기본적으로 비활성화되어 있습니다. 관리자는 사용자가 재정의할 수 없는 두 가지 관리 설정을 통해 가용성을 제어합니다:

설정목적미구성 시
channelsEnabled마스터 스위치. 모든 채널이 메시지를 전달하려면 true여야 합니다. claude.ai 관리 콘솔 토글 또는 관리 설정에서 직접 설정합니다. 꺼져 있으면 개발 플래그를 포함한 모든 채널을 차단합니다.채널 차단
allowedChannelPlugins채널이 활성화된 후 등록할 수 있는 플러그인. 설정 시 Anthropic이 관리하는 목록을 대체합니다. channelsEnabled가 true인 경우에만 적용됩니다.Anthropic 기본 목록 적용

조직이 없는 Pro 및 Max 사용자는 이러한 검사를 완전히 건너뜁니다: 채널을 사용할 수 있으며 사용자가 --channels로 세션별로 옵트인합니다.

조직에서 채널 활성화 ​

관리자는 claude.ai > Admin settings > Claude Code > Channels에서 채널을 활성화하거나, 관리 설정에서 channelsEnabled를 true로 설정할 수 있습니다.

활성화되면, 조직의 사용자가 --channels를 사용하여 개별 세션에 채널 서버를 옵트인할 수 있습니다. 설정이 비활성화되거나 미설정이면, MCP 서버는 연결되고 도구는 작동하지만 채널 메시지는 도착하지 않습니다. 시작 시 경고가 사용자에게 관리자에게 설정 활성화를 요청하라고 알려줍니다.

실행 가능한 채널 플러그인 제한 ​

기본적으로 Anthropic이 관리하는 허용 목록에 있는 모든 플러그인이 채널로 등록할 수 있습니다. Team 및 Enterprise 플랜의 관리자는 관리 설정에서 allowedChannelPlugins를 설정하여 자체 허용 목록으로 대체할 수 있습니다. 이를 사용하여 허용되는 공식 플러그인을 제한하거나, 자체 내부 마켓플레이스의 채널을 승인하거나, 둘 다 할 수 있습니다. 각 항목은 플러그인과 해당 마켓플레이스를 지정합니다:

json
{
  "channelsEnabled": true,
  "allowedChannelPlugins": [
    { "marketplace": "claude-plugins-official", "plugin": "telegram" },
    { "marketplace": "claude-plugins-official", "plugin": "discord" },
    { "marketplace": "acme-corp-plugins", "plugin": "internal-alerts" }
  ]
}

allowedChannelPlugins가 설정되면, Anthropic 허용 목록을 완전히 대체합니다: 나열된 플러그인만 등록할 수 있습니다. 기본 Anthropic 허용 목록으로 돌아가려면 미설정 상태로 두세요. 빈 배열은 허용 목록의 모든 채널 플러그인을 차단하지만, --dangerously-load-development-channels는 여전히 로컬 테스트를 위해 우회할 수 있습니다. 개발 플래그를 포함하여 채널을 완전히 차단하려면, 대신 channelsEnabled를 미설정 상태로 두세요.

이 설정은 channelsEnabled: true가 필요합니다. 사용자가 목록에 없는 플러그인을 --channels에 전달하면, Claude Code는 정상적으로 시작되지만 채널이 등록되지 않으며, 시작 알림에서 플러그인이 조직의 승인 목록에 없다고 설명합니다.

리서치 프리뷰 ​

채널은 리서치 프리뷰 기능입니다. 가용성이 점진적으로 출시되고 있으며, --channels 플래그 구문과 프로토콜 계약은 피드백에 따라 변경될 수 있습니다.

프리뷰 기간 동안 --channels는 Anthropic이 관리하는 허용 목록의 플러그인만 허용하거나, 관리자가 allowedChannelPlugins를 설정한 경우 조직의 허용 목록을 사용합니다. claude-plugins-official의 채널 플러그인이 기본 승인 세트입니다. 유효한 허용 목록에 없는 것을 전달하면, Claude Code는 정상적으로 시작되지만 채널이 등록되지 않으며, 시작 알림에서 이유를 알려줍니다.

개발 중인 채널을 테스트하려면 --dangerously-load-development-channels를 사용하세요. 직접 만든 커스텀 채널 테스트에 대한 자세한 내용은 리서치 프리뷰 기간 테스트를 참조하세요.

문제나 피드백은 Claude Code GitHub 저장소에 보고해 주세요.

채널 비교 ​

여러 Claude Code 기능이 터미널 외부의 시스템과 연결되며, 각각 다른 종류의 작업에 적합합니다:

기능하는 일적합한 용도
웹에서의 Claude CodeGitHub에서 복제된 새로운 클라우드 샌드박스에서 작업 실행나중에 확인할 독립적인 비동기 작업 위임
Slack에서의 Claude채널이나 스레드의 @Claude 멘션에서 웹 세션 생성팀 대화 맥락에서 직접 작업 시작
표준 MCP 서버작업 중 Claude가 쿼리. 세션에 푸시되지 않음Claude에게 시스템 읽기 또는 쿼리에 대한 온디맨드 접근 제공
Remote Controlclaude.ai 또는 Claude 모바일 앱에서 로컬 세션 제어자리를 비운 동안 진행 중인 세션 조종

채널은 이 목록의 빈 공간을 채웁니다: Claude가 아닌 소스에서 이미 실행 중인 로컬 세션으로 이벤트를 푸시합니다.

  • 채팅 브릿지: Telegram, Discord, iMessage를 통해 휴대폰에서 Claude에게 질문하면, 작업이 실제 파일에 대해 머신에서 실행되는 동안 같은 채팅에서 답변이 돌아옵니다.
  • 웹훅 수신기: CI, 오류 추적기, 배포 파이프라인 또는 기타 외부 서비스의 웹훅이 Claude가 이미 파일을 열고 디버깅하던 내용을 기억하고 있는 곳에 도착합니다.
