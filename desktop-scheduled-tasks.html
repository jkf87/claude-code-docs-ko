Claude Code Desktop에서 반복 작업 예약하기 ​

기본적으로 예약 작업은 사용자가 선택한 시간과 빈도에 맞춰 자동으로 새 세션을 시작합니다. 일일 코드 리뷰, 의존성 업데이트 확인, 캘린더와 메일함에서 정보를 가져오는 아침 브리핑 등 반복적인 작업에 활용하세요.

예약 옵션 비교 ​

Claude Code는 반복 작업을 예약하는 세 가지 방법을 제공합니다:

클라우드Desktop/loop
실행 위치Anthropic 클라우드사용자 머신사용자 머신
머신 켜져 있어야 함아니오예예
세션 열려 있어야 함아니오아니오예
재시작 후에도 유지예예아니오 (세션 범위)
로컬 파일 접근아니오 (새로 클론)예예
MCP 서버작업별로 구성된 커넥터설정 파일 및 커넥터세션에서 상속
권한 프롬프트아니오 (자율 실행)작업별 구성 가능세션에서 상속
사용자 정의 일정CLI에서 /schedule로 설정예예
최소 간격1시간1분1분

TIP

머신 없이도 안정적으로 실행되어야 하는 작업에는 클라우드 작업을 사용하세요. 로컬 파일과 도구에 접근해야 할 때는 Desktop 작업을 사용하세요. 세션 중 빠른 폴링에는 **/loop**을 사용하세요.

Schedule 페이지는 두 가지 종류의 작업을 지원합니다:

  • 로컬 작업: 사용자 머신에서 실행됩니다. 로컬 파일과 도구에 직접 접근할 수 있지만, Desktop 앱이 열려 있고 컴퓨터가 깨어 있어야 실행됩니다.
  • 원격 작업: Anthropic이 관리하는 클라우드 인프라에서 실행됩니다. 컴퓨터가 꺼져 있어도 계속 실행되지만, 로컬 체크아웃이 아닌 저장소의 새 클론에 대해 작업합니다.

두 종류 모두 동일한 작업 그리드에 표시됩니다. New task를 클릭하여 어떤 종류를 생성할지 선택하세요. 이 페이지의 나머지 내용은 로컬 작업에 대해 다룹니다. 원격 작업에 대해서는 클라우드 예약 작업을 참조하세요.

로컬 작업의 놓친 실행 및 보충 실행 동작에 대한 자세한 내용은 예약 작업 실행 방식을 참조하세요.

INFO

기본적으로 로컬 예약 작업은 커밋되지 않은 변경 사항을 포함하여 작업 디렉토리의 현재 상태에 대해 실행됩니다. 프롬프트 입력에서 worktree 토글을 활성화하면 병렬 세션과 동일한 방식으로 각 실행에 격리된 Git worktree를 제공합니다.

예약 작업 만들기 ​

로컬 예약 작업을 만들려면 사이드바에서 Schedule을 클릭하고, New task를 클릭한 다음 New local task를 선택하세요. 다음 필드를 구성합니다:

필드설명
Name작업의 식별자입니다. 소문자 케밥 케이스로 변환되어 디스크의 폴더 이름으로 사용됩니다. 작업 간에 고유해야 합니다.
Description작업 목록에 표시되는 간단한 요약입니다.
Prompt작업이 실행될 때 Claude에게 전송되는 지시사항입니다. 프롬프트 박스에 메시지를 작성하는 것과 동일한 방식으로 작성하세요. 프롬프트 입력에는 모델, 권한 모드, 작업 폴더, worktree 관련 컨트롤도 포함됩니다.
Frequency작업 실행 빈도입니다. 아래의 빈도 옵션을 참조하세요.

아무 세션에서든 원하는 내용을 설명하여 작업을 만들 수도 있습니다. 예를 들어, "매일 아침 9시에 실행되는 일일 코드 리뷰를 설정해줘."

빈도 옵션 ​

빈도 드롭다운에서 프리셋을 선택하거나, 피커에서 제공하지 않는 일정은 Claude에게 요청하세요:

  • Manual: 일정 없음, Run now을 클릭할 때만 실행됩니다. 필요할 때 트리거하는 프롬프트를 저장하는 데 유용합니다
  • Hourly: 매시간 실행됩니다. 각 작업은 API 트래픽을 분산하기 위해 정시로부터 최대 10분의 고정 오프셋을 갖습니다
  • Daily: 시간 선택기가 표시되며, 기본값은 현지 시간 오전 9:00입니다
  • Weekdays: Daily와 동일하지만 토요일과 일요일을 건너뜁니다
  • Weekly: 시간 선택기와 요일 선택기가 표시됩니다

피커에서 제공하지 않는 간격(15분마다, 매월 1일 등)의 경우, Desktop 세션에서 Claude에게 일정을 설정하도록 요청하세요. 자연어를 사용하세요. 예를 들어, "6시간마다 모든 테스트를 실행하는 작업을 예약해줘."

예약 작업 실행 방식 ​

로컬 예약 작업은 사용자 머신에서 실행됩니다. Desktop은 앱이 열려 있는 동안 매분 일정을 확인하고, 작업이 예정된 시간이 되면 수동 세션과 독립적으로 새 세션을 시작합니다. 각 작업은 예정 시간 이후 API 트래픽을 분산하기 위해 최대 10분의 고정 지연을 갖습니다. 이 지연은 결정적입니다: 동일한 작업은 항상 동일한 오프셋에서 시작됩니다.

작업이 실행되면 데스크톱 알림을 받고 사이드바의 Scheduled 섹션 아래에 새 세션이 나타납니다. 열어서 Claude가 수행한 작업을 확인하거나, 변경 사항을 검토하거나, 권한 프롬프트에 응답할 수 있습니다. 세션은 다른 세션과 동일하게 작동합니다: Claude는 파일을 편집하고, 명령을 실행하고, 커밋을 생성하고, 풀 리퀘스트를 열 수 있습니다.

작업은 Desktop 앱이 실행 중이고 컴퓨터가 깨어 있는 동안에만 실행됩니다. 예정된 시간에 컴퓨터가 절전 모드에 있으면 실행이 건너뛰어집니다. 유휴 절전을 방지하려면 설정의 Desktop app → General에서 Keep computer awake를 활성화하세요. 노트북 덮개를 닫으면 여전히 절전 모드로 전환됩니다. 컴퓨터가 꺼져 있을 때도 실행해야 하는 작업에는 원격 작업을 사용하세요.

놓친 실행 ​

앱이 시작되거나 컴퓨터가 깨어나면, Desktop은 지난 7일 동안 각 작업의 놓친 실행이 있는지 확인합니다. 있다면, Desktop은 가장 최근에 놓친 시간에 대해 정확히 한 번의 보충 실행을 시작하고 그보다 오래된 것은 버립니다. 6일을 놓친 일일 작업은 깨어났을 때 한 번 실행됩니다. Desktop은 보충 실행이 시작될 때 알림을 표시합니다.

프롬프트를 작성할 때 이점을 염두에 두세요. 오전 9시로 예약된 작업이 컴퓨터가 하루 종일 절전 모드였다면 오후 11시에 실행될 수 있습니다. 시간이 중요하다면 프롬프트 자체에 가드레일을 추가하세요. 예를 들어: "오늘의 커밋만 검토해줘. 오후 5시 이후라면 리뷰를 건너뛰고 놓친 것의 요약만 게시해줘."

예약 작업 권한 ​

각 작업에는 자체 권한 모드가 있으며, 작업을 만들거나 편집할 때 설정합니다. ~/.claude/settings.json의 허용 규칙도 예약 작업 세션에 적용됩니다. 작업이 Ask 모드에서 실행되고 권한이 없는 도구를 실행해야 하는 경우, 승인할 때까지 실행이 중단됩니다. 세션은 사이드바에 열려 있어 나중에 응답할 수 있습니다.

중단을 방지하려면 작업을 만든 후 Run now을 클릭하고 권한 프롬프트를 확인한 다음, 각각에 대해 "always allow"를 선택하세요. 해당 작업의 향후 실행에서는 프롬프트 없이 동일한 도구를 자동 승인합니다. 작업의 세부 페이지에서 이러한 승인을 검토하고 취소할 수 있습니다.

예약 작업 관리 ​

Schedule 목록에서 작업을 클릭하면 세부 페이지가 열립니다. 여기에서 다음을 수행할 수 있습니다:

  • Run now: 다음 예정 시간을 기다리지 않고 즉시 작업을 시작합니다
  • Toggle repeats: 작업을 삭제하지 않고 예약 실행을 일시 중지하거나 재개합니다
  • Edit: 프롬프트, 빈도, 폴더 또는 기타 설정을 변경합니다
  • Review history: 컴퓨터가 절전 모드여서 건너뛴 것을 포함하여 모든 과거 실행을 확인합니다
  • Review allowed permissions: Always allowed 패널에서 이 작업에 대해 저장된 도구 승인을 확인하고 취소합니다
  • Delete: 작업을 제거하고 생성된 모든 세션을 아카이브합니다

Desktop 세션에서 Claude에게 요청하여 작업을 관리할 수도 있습니다. 예를 들어, "dependency-audit 작업을 일시 중지해줘", "standup-prep 작업을 삭제해줘", "내 예약 작업을 보여줘."

작업의 프롬프트를 디스크에서 편집하려면 ~/.claude/scheduled-tasks/<task-name>/SKILL.md를 열어보세요 (설정된 경우 CLAUDE_CONFIG_DIR 아래). 파일은 name과 description에 대한 YAML frontmatter를 사용하며, 본문이 프롬프트입니다. 변경 사항은 다음 실행에 적용됩니다. 일정, 폴더, 모델, 활성화 상태는 이 파일에 없습니다: Edit 양식을 통하거나 Claude에게 요청하여 변경하세요.
